Code First

  • Model is defined in code using C# classes. From this, EF will create database objects and automatically provide the mapping between the two
    • Additional options can be configured using attributes on the classes

Setting Up with No Database

  • Create a new context class that derives from the System.Data.Entity.DbContext class
    public class BloggingContext : DbContext
  • For each entity, expose a DbSet object in the context
    public DbSet<Post> Posts { get; set; }
  • Data annotations can be added to the model
    The following sets the Username property to be the primary key of the User table
    public class User
    {
    [Key]
    public string Username { get; set; }
    }
