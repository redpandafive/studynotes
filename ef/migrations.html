Migrations

  • When the model changes, use migrations to send the changes to the database
  • Allows an ordered set of steps that describe how to upgrade our database schema
  • Running the enable-migrations command from the Package Manager Console does the following:
    • Creates a new Migrations folder
    • Adds a Configuration.cs file to the Migrations folder
    • Adds a timestamp_InitialCreate.cs file to the Migrations folder
      • Contains the steps needed to take an empty database to one that matches the model

Configuration.cs

  • Contains the settings that migrations will use, including
    • Seeding data
    • Register providers for other databases

New Migrations

  • To create a new migration after a model change, use the Add-Migration migration-name
  • This checks for model changes since the last migration and scaffolds a new migration with any changes that are dound

Modifying the Database

  • Use the Update-Database command to apply any pending migrations to the database

Code Description

  • Migrations are classes generated by EF
  • Each as two methods
    • Up = code that will be executed when you run this migration against the database
    • Down = code that will be executed when you rollback this migration

Migration History

  • When you execute a migration against a database, a _EFMigrationsHistory table is created
  • This contains a history of migrations that have been run against the databse
