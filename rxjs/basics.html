Basics

  • Reactive Extensions for JavaScript
  • "A library for composing asynchronous an event based programs by using observable sequences"
    • Provides a way to observe and react to data as it flows through time and the application
    • Data is emitted over time and can be transformed, filtered, modified, combined, and cached
  • This is one method for handling asynchronous data flow. Over methods include
    • Callbacks - a function called when data is returned
    • Promises - an object that will return a single data set from an asynchronous operation. It is not cancellable.
    • Async / await
      • Syntax for writing asynchronous code
      • The code looks synchronous
      • Can only handle a single emission
      • Not cancellable
  • Benefits to RxJS over other asynchronous methods:
    • The same techniques and operators work with a variety of data sources, including HTTP calls, mouse events, etc
    • Easily allows composition of data from multiple data sources
    • Can product multiple values over time
    • Lazy evalaution - does not start processing data streams until subscription
    • Handles errors with built in operators
    • Asynchronous actions are cancellable
  • Angular was build aroundRxJS and observables
    • Many features use observables to watch for changes, including:
      • Route changes and updates
      • Reactive forms and changes
      • HttpClient for HTTP communication
      • Allows a more reactive style programming (see Rective Programming)
