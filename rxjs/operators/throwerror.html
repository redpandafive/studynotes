ThrowError

  • Creates an observable that immediately emits an error notification
  • It does not emit any items
  • Useful for testing error handling in observables by creating observables that always fail
  • Often, it is more useful to use the JS throw statement

Examples

  • This will return EMPTY values when the stream fails
  • This will fix the broken stream by substituting and EMPTY observable
this.http.get<DataObject>("http://mytargeturl").pipe(
catchError(() => EMPTY)
)
This example catches an error, logs it to the console, then throws a new error
this.http.get<DataObject>("http://mytargeturl").pipe(
catchError(err => {
console.log(err);
return throwError(() => new Error("could not retrieve"));
})

)
Usually simpler to just use the JS throw function
this.http.get<DataObject>("http://mytargeturl").pipe(
catchError(err => {
console.log(err);
throw new Error("could not retrieve");
})

)
Another option is to set a property on the component to display an error message, then return EMPTY
this.products$ = this.productService.getProducts().pipe(
catchError(err => {
this.errorMessage = err;
return EMPTY;
})

)
