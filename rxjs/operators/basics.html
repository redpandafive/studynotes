Operator Basics

  • Operators are functions used to transform and manipulate emitted items
    • For each item emitted, the value passed through a set of operators in sequence using the Observable's pipe method
    • Often, these take an arrow function as a parameter
    • For each emitted item, the value from the last operator in the pipeline is emitted by the Observable's next method
  • These were fashioned after LINQ in .NET
  • These take an Observable and return an Observable
    • This allows you to chain operators together to perform complete transformations
    • The Input Observable is the observable object passed to an operator
    • Operators will automatically subscribe to their input observable
    • Operators create and return a new observable called an Output Observable
      • This is passed to the next operator in the pipeline
    • This process repeats for each operator in the pipeline
  • There are around 100 operators in RxJS

Marble Diagrams

  • Describes an RxJS operator
  • The lines represent the flow of time
    • The top line represents the Source Observable to the operator
    • The bottom line represents the Output Observable to the operator
  • Each emitted item is represented by a marble on the timeline
  • A straight vertical line represents a complete notification (for operators that automatically complete)
  • An x on a line represents an error
