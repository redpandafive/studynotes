Timer

  • Returns an observable that emits after the specified delay
  • Useful for
    • Creating delays in code
    • An alternative to interval when you want to start right away
    • Delaying subscription to another observable by a set amount of time
  • Parameters
    • 1st parameter is the number of milliseconds to wait before emitting
      • This is optional; default is 0
    • 2nd parameter is interval to wait before emitting again
      • This is optional
      • If not defined, the observable will only emit once
      • This will continue to emit values until the Observable is stopped
Example that:
  • Delays first emission by 1 second
  • Emits 0
  • Waits 5 seconds
  • Emits 1
  • It will continue to do this until the Observable is stopped
timer(1000, 5000).subscribe(console.log);
Example that:
  • Emits 0 (no delay)
  • Waits 5 seconds
  • Emits 1
  • It will continue to do this until the Observable is stopped
timer(0, 5000).subscribe(console.log);
This example:
  • Delays 3 seconds
  • Subscribes to the source$ observable
  • Emits each integer from the observable without delays between emissions
const source$ = of(1, 2, 3);

timer(3000)
.pipe(concatMap(() => source$))
.subscribe(console.log);
