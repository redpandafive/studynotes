Share

  • Similar to shareReplay, but does not have a buffer
    • Share takes a configuration object, so it allows for more granular control over its behavior that shareReplay
  • Since it does not have a buffer, it does not replay a buffer to late subscribers
    • This can be modified by passing in an optional configuration object
  • For a simple caching setup, the shareReplay is a sufficient alternative
