Route Parameter / Input Mapping

WARNING: Requires Angular 16+

  • Previous way to read route parameters involved something like this
    constructor(private route: ActivatedRoute) {}
    ...
    ngOnInit() {
    const id = this.route.snapshot.paramMap.get('id');
    }
  • Route parameters can now be mapped to input properties on components
    • First, enable Input decorators mapping
      This uses standalone bootstrapping
      const appRoutes: Routes = [];
      bootstrapApplication(AppComponent, {
      providers: [
      provideRouter(appRoutes, withComponentInputBinding())
      ]
      })
      This uses classic bootstrapping
      const routes: Routes = [];

      @NgModule({
      imports: [RouterModule.forRoot(
      routes,
      { bindToComponentInputs: true }
      )],
      exports: [RouterModule]
      })
      export class AppRoutingModule { }
    • Add an input to the component that will receive the paramter value
      This imports a route paramter named id
      @Input() id = '';
      This imports a route paramter named id, but renames it to productId
      @Input('id') productId = '';
  • Route input paramters will be type string

Additional Options

  • Additional options were added to the input parameter that may be useful here, particularly the transform property.
