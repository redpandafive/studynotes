Basics

  • Newer versions of Angular use HTML History API strategy
  • Older versions separated Angular parts of the URL using the #

HTML Page Setup

  • Index.html page should have a base element added as the first child in the head tag to tell the router how to compose navigation URLs
  • If in the application root, you would add <base href="/">

Router Outlet Directive

  • Directive that is used like a component
  • A placeholder that marks the spot in a template where the router should display the components for that outlet
<router-outlet></router-outlet>

Routing with Mulitple Modules

  1. Give each feature module its own module file
  2. Give each fature module a component to act as the hub
    • This will contian the router-outlet element
  3. Create routing module in each feature module
    • This will hold routes specific to that module
  4. Import each routing module into its corresponding feature module
  5. For consistency, add a routing module to the root module
  6. Import each feature module into the root module
