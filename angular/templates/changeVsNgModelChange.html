Change vs NgModelChange

  • (change)
    • This is a DOM event - listens for a DOM event called change
    • Only fires when the user has blurred the input
      • This means it fires after the value bound to NgModel has changed
    • A regular Event object is passed to the handler when called
      • This is a plain DOM Event object
      • Need to use e.target.value to get the value
  • (ngModelChange)
    • This is an Angular event, so it is inside the Angular system
    • This fires when ngModel changes
      • Actually an event emitter on teh NgModel directive
      • This means you cannot it without ngModel
    • This will be fired when the model is intended to be updated (not waiting for a blur event, for example)
      • Means it will fire before teh value bound to NgModel has changed
    • The event handler will be passed just the value emitted, not the value wrapped in a DOM event object (you can name this whatever you want in the template)
