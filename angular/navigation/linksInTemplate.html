Defining Links in Templates

  • RouterLink directive on tags give the router control over those elements
    • navigation paths are fixed, so they are one-time bindings
  • QueryParams directive is used to pass query parameters
    • http://localhost:4200/books?orderby=price
    • Example
      <a [routerLink]="['/books']" [queryParams]={ orderby: 'price' }">"
    • QueryParamsHandling option can configure query parameter behavior
      • See Routing â€“ Navigating in Code section for details
    • Examples
      • code='<a routerLink="">Home</a>'
        • This creates a link to the root of the application
        • This would hit the empty route if defined
      • code='<a [routerLink]="courses">Courses</a>'
        • This creates a link that appends /courses onto the current URL
      • <a [routerLink]="['/events', event.id]">"
        • This creates a link to the events route and concatenates id onto the URL as a route parameter
      • <a [routerLink]="['/events']" [queryParams]="{ orderby: 'price', category: 'fiction' }" ..."
        • Adds ?orderby=price&category=fiction as a query parameter
