Styling Components

  • Use to isolate CSS rules to within the component template
    • Will not be inherited by any components nested within the template nor by any content projected into the components
    • By default, component CSS styles are encapsulated into the components view and don’t affect the rest of the application
  • Best practices
    • Do not override the styles of a component – consider then private implementation details for that component
    • If you are authoring a component that others will use, you can accept customization from consumers in a number of ways (see https://angular.io/guide/component-styles#authoring-a-component-to-support-customization)
  • Special selectors from the world of shadow DOM style scoping
    • :host
      • Host element – element that matches the components selector into which the template is rendered
      • This selector targets that host element
      • Rules will affect the host element and all its descendants
      • You can add selectors after the host selector to target elements within the component
    • :host(.active)
      • Function form of the host selector
      • Applies host styles conditionally
      • Includes another selector inside parentheses after :host
      • Example above targets the host element but only when it also has the active CSS class applied to it
    • :host-context
      • Use to apply styles based on some condition outside of a component’s view
      • :host-context(.theme-light) h2 would apply rules to all h2 elements inside the component only if some ancestor element has the CSS class theme-light
  • Ways to add styles for a component
    • By setting styles or styleUrls properties of the Component decorator
      • These are arrays or strings
      • If you are using the CLI to build, you can add sass, less, or stylus files into the styleUrls metadata with the appropriate extensions
      • Angular CLI build process runs the pertinent CSS preprocessor
    • Inline in the template HTML
      • Embed a style tag into the template or a link tag to reference an external stylesheet
  • With CSS imports
    • use to import CSS sheets into other CSS sheets
    • the URL is relative to the CSS file into which you are importing
  • To control how CSS encapsulation works on a component level, you can set the view encapsulation mode in the component metadata (encapsulation tag)
    • ViewEncapsulation.ShadowDom
      • Uses browser's native shadow DOM implementation. The component styles are included within the shadow DOM
      • Only works on browsers that have native support for shadow DOM, which is limited
    • ViewEncapsulation.Emulated
      • Default
      • Emulated the behavior of shadow DOM by preprocessing the CSS code to effective scope the CSS to the component's view
    • ViewEncapsulation.None
      • no view encapsulation. This behaves the same as pasting the component's styles into the HTML
  • If you need to apply styles across the entire page, add the rules to the site.CSS file

@import './hero-details-box.css'"
