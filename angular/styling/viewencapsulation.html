View Encapsulation

  • Use to isolate CSS rules to within the component template
    • Will not be inherited by any components nested within the template nor by any content projected into the components
    • By default, component CSS styles are encapsulated into the components view and donâ€™t affect the rest of the application
  • To control how CSS encapsulation works on a component level, you can set the view encapsulation mode (encapsulation tag)
    • This is configured in the component metadata
      ... encapsulation: ViewEncapsulation.None ...
    • ViewEncapsulation.ShadowDom
      • Uses browser's native shadow DOM implementation. The component styles are included within the shadow DOM
      • This shuts off Angular's emulated scope and encapsulation, so it is not emulated
      • Only works on browsers that have native support for shadow DOM, which is limited
    • ViewEncapsulation.Emulated
      • Default mode
      • Provides emulated scope and encapsulation
      • Emulated the behavior of shadow DOM by preprocessing the CSS code to effective scope the CSS to the component's view
      • Angular dynamically adds a custom scoping attribute to all nodes in the component
        • These are unique to a component
        • These are different each time Angular renders the component, so don't try to directly reference them
      • Component stylesheets are inserted as style blocks in the head of the page
        • Style rules are rewritten to add the custom attributes that Angular generated
    • ViewEncapsulation.None
      • This shuts off Angular's emulated scope and encapsulation
      • No view encapsulation. This behaves the same as pasting the component's styles into the HTML
      • Component styles are still inserted into the head element as written, but do not use any scoping to restrict their effect to a single component
      • Does have some use cases
        • Set global styling on the app component so they are applied to all children
          • This is inserted first, so all component stylesheets will follow and override the rules
