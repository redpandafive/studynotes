Shadow DOM Style Scoping Selectors

  • Angular emulates a handful of CSS selectors from the CSS scoping module
  • :host pseudo-class
    • Host element = element that matches the component's selector
      • Component's template is rendered within this element
    • This selector targets that host element
    • Rules will affect the host element and all its descendants
    • You can add selectors after the host selector to target elements within the component
    • This behavior is from Angular, so it can be used in older browsers that do not support this pseudo class
  • :host(.active)
    • Function form of the host selector
    • Applies host styles conditionally by:
      • Includes another selector inside parentheses after :host
      • This checks the host element for the condition
        • Example above targets the host element but only when it also has the active CSS class applied to it
    • Use this when you want to keep the search confined to the host element
  • :host-context
    • Use to apply styles based on some condition outside of a componentâ€™s view
    • This will check any parent element from the host all the way up to the document root to check for a match (as well as the host itself)
    • :host-context(.theme-light) h2 would apply rules to all h2 elements inside the component only if some ancestor element has the CSS class theme-light
    • Under the hood, it does this by adding two selectors to cover two cases
      • One to check if the class is added directly on the host
      • One to check if the host is nested within an element that has this class
    • Use this when you want to search up through the ancestry tree, which may not be desireable in some situations
  • ::ng-deep
    • Note, this has been deprecated in favor of native CSS
    • Allows you to pierce through the boundaries of the shadow DOM in child components (projected content)
    • Under the hood, it emulates this selector by just removing the ::ng-deep
      • This means they become global styles, so use with care
      • Probably better to use ::host ::ng-deep a to confine the styles to just the component and its children
    • Examples
      • ::ng-deep a
        • Checks child components an applies the CSS rules to any link tags
      • ::ng-deep a:hover
        • Checks child components an applies the CSS rules to any link tags when hovering over them
