Form Change Detection

  • FormGroups, FormArrays, and FormControls extend the AbstractControl base class
  • This means they have a valueChanges method that returns an observable
  • You can subscribe to it to respond when values change
  • This event is triggered immediately after the new values are updated, but before the change is bubbled up to its parent
    • this.formGroup.value will still show the previous value
  • For ForGroups and FormArrays, the event is fired any time a child control value changes

Setups

Given a FormGroup parentForm with a control firstname, add the following to listen for control changes
this.parentForm.get("firstname").valueChanges.subscribe(x => {
console.log('firstname value changed');
console.log(x);
})
Given a FormGroup parentForm, add the following to listen for form changes
this.parentForm.valueChanges.subscribe(newVal => {
console.log(newVal);
})
The event is raised even if the value is changed programmatically. Sometimes you want to supress the event (suchs as changing it programmatically):
this.parentForm.get("firstname").setValue("", { emitEvent: false });
Events usually will bubble from a FormConntrol up to its parent FormGorup. You can isolate the event to just the control and prevent bubbling with the following:
this.parentForm.get("firstname").setValue("", { onlySelf: true });
Template driven forms use the same syntax in the component, but need a reference to the Form Model with the following:
@ViewChild('templateForm',null) templateForm: NgForm;
