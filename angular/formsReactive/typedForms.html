Typed Reactive Forms

Basics

  • Starting with Angular 14, reactive forms are strictly typed by default
    • This does not apply to template forms
  • One approach is to use inference of the initial value define typing
    • The following FormControl is typed to string | null
      • It is typed to null since that is the value it will go to when reset
      const email = new FormControl('myEmail@gmail.com');
    • A non-nullable version of the above declaration would be:
      • When reset, its value will be myEmail@gmail.com
      const email = new FormControl('myEmail@gmail.com', {nonNullable: true});
  • To set an explicit type
    const email = new FormControl(null);

FormArray

  • FormArray type parameters correspond to the type of each inner control
  • If you wnat a mixture of types within the same FormArray you must use UntypedFormArray

Migration

  • To migrate, Untyped versions of the controls were added, such as UntypedFormGroup and UntypedFormControl
    • These are compatible with previous Angular versions
