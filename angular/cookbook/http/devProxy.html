Proxy to Fix CORS Problems in Development

  • While the Angular app and backend may be running on the same domain in production, they may be running on seperate domains locally during Development
  • This can cause CORS policy violations that prevent HTTP comminication
  • A proxy can be used to fix this in development so modifications do not need to be made to the backend code to allow communication
>

Steps to Implement

In the src folder, create a file called proxy.conf.json with the following contents:
{
"/api": {
"target": "http://localhost:3001",
"secure": false
}
}
In angular.json file for the project, add a refrence to the proxy config file. This should go under:
  • projects
  • project_name
  • architect
  • serve
  • options (may need to add this item)
..
"options": {
"proxyConfig": "src/proxy.conf.json"
}
Restart the server and client for this to take effect. The following is an example URL that you would reference when calling the backend:
/api/Reasons/GetByReasonSet/5
Make sure to check the mapping on server side for a route that begins in "api"
