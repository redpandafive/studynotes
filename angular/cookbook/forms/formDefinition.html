Basic Form with No FormGroup

Register the reactive forms module to include the reactive form directives needed
import { ReactiveFormsModule } from '@angular/forms';
Add a FormControl instance to the component
export class NameEditorComponent {
name = new FormControl('');
}
Register the FormControl in the template
<input id="name" type="text" [formControl]="name">

Form Using a FormGroup

Create a FormGroup instance on the component
  • Provide the constructor with an object of named keys mapped to their control
profileForm = new FormGroup({
firstName: new FormControl(''),
lastName: new FormControl(''),
});
Associate the FormGroup model and view.
<form [formGroup]="profileForm"> <label
for="first-name">First Name: </label> <input
id="first-name" type="text" formControlName="firstName">
<label for="last-name">Last Name: </label> <input
id="last-name" type="text" formControlName="lastName">
</form>
The FormGroup directive listens for the submit event emitted by the form element and emits an ngSubmit event that you can bind to a callback function. Add an ngSubmit event listener to the form tag with the onSubmit() callback method.
<form [formGroup]="profileForm" (ngSubmit)="onSubmit()">

Splitting Forms Across Subcomponents

Approach 1: Pass the FormGroup as Input

Approach 2: Use the FormGroupDirective

  • A child component can have access to the parent component's FormGroup instance. It does this through the FormGroupDirective
    • This is injected into the constructor of the child component
    • Access this in the OnInit method
    • This gives access to the parent's FormGroup instance
Define the following in the child component. No additional configuration is needed in the parent component.
form: FormGroup;
...
ngOnInit(): void {
this.form = this.rootFormGroup.control;
}
Define the following in the child template.
<form [formGroup]="form">
...
ngOnInit(): void {
this.form = this.rootFormGroup.control;
}

Handling FormGroups in Split Forms

  • The root FormGroup in the parent may divide controls into one or more sub-FormGroups
    this.userForm = this.fb.group({
    basicInfo: this.fb.group({
    firstName: [],
    lastName: [],
    email: [],
    age: []
    }),
    address: this.fb.group({
    street: [],
    number: [],
    postal: [],
    company: []
    })
    });
  • The child template should reference the FormGroup needed since it has access to the entire FormGroup
    <form [formGroup]="form">
    <div formGroupName="basicInfo">
    ...
  • Alternatively, you can isolate the nested FormGroup and leave out the reference in the template
    ngOnInit(): void {
    this.form = this.rootFormGroup.control.get("basicInfo") as FormGroup
    }
  • This can be made more flexible by passing the FormGroup name as in as an input
    // in the parent template
    <app-basic-info formGroupName="basicInfo"></app-basic-info>
    ..
    // in the child component
    @Input() formGroupName: string;
    ...
    ngOnInit(): void {
    this.form = this.rootFormGroup.control.get(this.formGroupName) as FormGroup;
    }
