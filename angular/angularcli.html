Angular CLI

  • Commands
  • Gotchas

Commands

npm install -g @angular/cli
  • Installs Angular CLI globally from npm
ng new my-app
  • Creates a workspace and initial application in a folder titled my-app
  • Creates initial skeleton application in src folder
ng serve –open
  • Runs the application using a local server (assuming you are in a project folder)
  • Launches server, watches files, and rebuilds the app as you make changes to files
  • The –open option automatically opens the browser to http://localhost:4200
ng generate component heroes/hero-list
  • Creates a new component named heroes, including the HTML, CSS, component Typescript file, and a Typescript spec file to use for testing
  • Adds the component to the module heroes automatically
ng generate service hero
  • This will register this new provider with the root injector
ng generate module orders –routing
  • Makes a new folder called orders containing an OrdersModule and an OrdersRoutingModule
ng generate module app-routing --flat --module=app
  • The –flat switch puts the file in the src/app instead of its own folder
  • The –module=app tells the CLI to register it in the imports array of the AppModule
ng generate pipe
  • Creates a custom pipe class and registers the pipe in the declarations section of NgModules automatically
ng lint
  • The first time this is run, all necessary packages will be installed
  • After installation, running this will lint the entire project
  • Adding the --fix parameter will fix all fixable items
