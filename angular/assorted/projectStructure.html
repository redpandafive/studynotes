Project Structure

  • Angular loads as a collection of JS modules (think of them as library modules)
  • Each library begins with the @angular prefix
    • Use NPM to install these
  • Import parts of them with JS import statements
    • This can be done to import components as well as modules
  • Workspace
    • Contains the files for one or more projects
    • Created by the new command in CLI
    • Commands that create or operate on apps and libraries must be executed from within a workspace folder
    • All apps in a workspace can use libraries in the same workspace
  • Project
    • Set of files that comprise an app, a library, or end to end tests
  • Use Angular CLI to create a new project skeleton from scratch
    • E2e folder
      • Testing folder
    • Node_modules folder
      • Packages from NPM
    • Src folder
      • Contains source code
        • App folder - where you will do most of your coding
      • Assets folder
        • Site wide static assets, such as images and site wide CSS
        • Can separate these into separate sub folders
        • Reference these as “/assets/mypic.png”
        • Assets folder path is referenced in the angular.json
          • Take arrays in assets, styles, and scripts keys
          • Use these to tell webpack to include these files
    • Angular.json
      • Various settings that the cli will use for various operations, like Web Pack fill
      • Global stylesheets can be registered in the style section (includes styles.css by default)
    • Package.json - NPM packages that should be downloaded with this project as well as defined scripts
    • Package-lock.json
