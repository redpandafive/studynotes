Angular .vs Angular JS

  • Can bind to the DOM properties, which eliminates 40+ directives from Angular 1
  • Components identify their template that they will use
  • In Angular 1 you would usually associate a controller with a template in the router

Ahead-of-Time Compilation

  • The HTML-like templates of Angular components are a challenge for external tools to analyze
    • Makes tree shaking less efficient since they cannot examine templates to see if some imports should be excluded
    • AoT compiler transpiles the Angular HTML-like templates to Javascript or Typescript with ES2015 module imports
      • This allows tree shaking to function correctly for imports used in templates

Project Structure

  • Angular loads as a collection of JS modules (think of them as library modules)
  • Each library begins with the @angular prefix
    • Use NPM to install these
  • Import parts of them with JS import statements
    • This can be done to import components as well as modules
  • Workspace
    • Contains the files for one or more projects
    • Created by the new command in CLI
    • Commands that create or operate on apps and libraries must be executed from within a workspace folder
    • All apps in a workspace can use libraries in the same workspace
  • Project
    • Set of files that comprise an app, a library, or end to end tests
  • Use Angular CLI to create a new project skeleton from scratch
    • E2e folder
      • Testing folder
    • Node_modules folder
      • Packages from NPM
    • Src folder
      • Contains source code
        • App folder - where you will do most of your coding
      • Assets folder
        • Site wide static assets, such as images and site wide CSS
        • Can separate these into separate sub folders
        • Reference these as “/assets/mypic.png”
        • Assets folder path is referenced in the angular.json
          • Take arrays in assets, styles, and scripts keys
          • Use these to tell webpack to include these files
    • Angular.json
      • Various settings that the cli will use for various operations, like Web Pack fill
      • Global stylesheets can be registered in the style section (includes styles.css by default)
    • Package.json - NPM packages that should be downloaded with this project as well as defined scripts
    • Package-lock.json

Bootstrapping

  • Loads main.ts file
    • This is referenced by the main property of angular.json to identify the startup
    • Enables Prod mode if that is enabled
    • Bootstraps AppModule with a call to platformBrowserDynamic().boostrapModule(AppModule)
  • AppModule bootstraps AppComponent
    • This is defined as a property on the NgModule decorator under the boostrap configuration property
    • Foundation component for an Angular project, so all applications require one and only one
  • App component
    • When you bootstrap with the AppComponent class (in main.ts), Angular looks for a app-root in the index.html, instantiates an instance of AppComponent, and renders it inside the app-root tag
    • The app-root tag will be in the Index.html file included in the root of the Src folder
  • Index.html
    • Loaded by the browser when the application starts
    • Contains the app-root tag from Angular

Import / Export Statements

You can alias imports as follows (within your script, you would reference this as TOASTR_TOKEN2)

import { TOASTR_TOKEN as TOASTR_TOKEN2 } from './common/toastr2.service'

Import/Export Statements – Simplifying

You can shorten the import declarations you need to reference items in feature modules by re-exporting the components in your feature module (AKA creating barrels)

To implement:
  • Put in index.ts file in the root of your feature module (this is the convention used in the Pluralsight course)
  • Re-export all the components in the module in this file
export * from './create-event.component';
export * from './event-thumbnail.component';
export * from './events-list-resolver.component';

Create a new index.ts in the tier above this and export all from the child index.ts files in this one file. Reference this outermost index.ts to perform all inputs

import {  EventsListComponent,  EventListResolver,  EventThumbnailComponent } from './events/index'

Views

  • Combination of a component and its template
    • Component controls the patch of screen called a view
  • Host view
    • Template immediately associated with a component
    • This can be part of a view hierarchy
  • Embedded views
    • Hosted views from other components that may be used to build up a component view
  • View Hierarchy
    • Composition of views that define arbitrarily complex areas of the screen that can be created, modified, and destroyed as a unit
    • This can mix views defined in components that belong to different Modules. This is especially true with UI libraries

Design Patterns

  • Observable Data Service
    • A simple and intuitive pattern
      • Useful when you have several components that need to reference the same data
    • Create a new service for the data store
    • In the constructor,
      • Inject the service that fetches the data from the server
      • Call the method on the injected service to fetch the data
    • Add a private property to capture a BehaviorSubject
      • This also provides an empty starting value
        private _todos: BehviorSubject<List<Todo>> = new BehaviorSubject(List([]))
    • In the constructor, call the service to hydrate the data
    • Don't expose the BehaviorSubject directly to clients
      • Expose an Observable instead
        • Prevents service clients from emitting store values directly and bypassing the store
        • Prevents event chaining in components that can be hard to understand
        • Exposes too much of the internal structures of the service needlessly

Angular Elements

  • Angular components packaged as custom elements, a web standard for defining new HTML elements in a framework agnostic way
  • Extends HTML by allowing you to define a tag whose content is created and controlled by JavaScript code
  • The browser maintains a CustomElementRegistry of defined custom elements which map an instantiable JavaScript class to an HTML tag
  • Currently supported by Chrome, Firefox, Opera, and Safari
    • Polyfills available through other browsers
  • Transforming a component to a custom element makes all of the required Angular infrastructure available to the browser
    • a small, self contained version of the Angular framework will be injected as a service to support the component's change detection and data-binding functionality
  • Custom elements
    • Bootstrap themselves automatically when they are added to the DOM
    • automatically destroyed when removed from the DOM
    • looks and behaves like any other HTML element
    • does not require any special knowledge of Angular terms or usage conventions
  • How it works
    • Use the createCustomElement function to convert a component into a class that can be registered with the browser as a custom element
      • once registered, it can be used just like a built in HTML element int content that you add directly into the DOM
      • this function collects the component's observable properties, along with the Angular functionality the browser needs to create and destroy the instances, as well as to detect and respond to changes
    • When the custom element is placed on a pace, the browser creates and instance of the registered class and adds it to the DOM
      • content is provided by the component template, which uses Angular template syntax
      • input properties in the component correspond to input attributes for the element

  • There are several methods on this service to scrolling
