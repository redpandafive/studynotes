Signals

  • Wrapper around a value that can notify interested onsumers when that value changes
  • Can contain nay value, from simple primitives to complex data struture
  • When the signal is updated, anything that depends on it will also be updated
    • Kind of like a BehviorSubject in RxJS, but you do not have to subscribe to get a value out
  • These can replace Observables
    • Simpler since you do not have to subscribe to them
    • Handle synchronous reactivity well (value is known and can be set immediately)
    • Does not handle asynchronous reactivity well (need to wait some time, such as a HTTP request)

Writable Signals

  • signal()
    • Use to create a writable signal (creates a WriteableSignal object)
    • These are getter functions, so calling them reads thier value
      const count = signal(0);
      console.log('The count is: ' + count());
  • Set()
    • Use to change the value of a writable signal
      count.set(3);
  • Update()
    • Use to compute a new value from the previous one
      count.update(value => value + 1);
  • Mutate()
    • When a signal contains an object, use this to mutate the object directly
    • The following changes the first object in the array to 'done: true' without replacing it.
      const todos = signal([{title: 'Learn signals', done: false}]);
      todos.mutate(value => {
      value[0].done = true;
      });
