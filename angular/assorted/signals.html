Signals

  • Zero argument functions. When executed, they return the current value of the signal
  • When the signal is updated, anything that depends on it will also be updated
    • Kind of like a BehviorSubject in RxJS, but you do not have to subscribe to get a value out
  • These can replace Observables
    • Simpler since you do not have to subscribe to them
    • Handle synchronous reactivity well (value is known and can be set immediately)
    • Does not handle asynchronous reactivity well (need to wait some time, such as a HTTP request)
  • Signals may be able to be converted to an Observables
    • Allows you to start with Signals, but convert to an Observable if you need to
  • One area this may change Angular is change ChangeDetection
    • Before
      • Zone.js detects that something might have changed
        • Zone.js patches browser APIs so that anytime anything that could possibly change the template, it will notify Angular that something changed
        • These could be DOM events, settimeout occured, etc
      • This triggers Angular change detection
      • Entire component tree is checked for changes
        • Sounds inefficient, but Angular handles it efficently
        • Still a heavy handed approach to change detection
    • After
      • Change is detected on a specific item (no Zone.js)
      • Angular change detection is run
      • Views are updated for anything that was changed
