Unidirectional Data Flow

  • A programming pattern that deals with how data is updated
  • There are two similar Patterns
    • Unidirectional - data can only flow in one direction
    • Bidirectional - data can flow in two directions
  • Angular uses an unidirectional data flow refers to pass data from a parent component to a child component
    • These bindings are updated during change detections - this is implemented via input bindings
    • When a bound property on a parent component is updated, those changes are also reflected in the child component bound property
    • This allows a top-down pass during a change cycle. State will be stabilized after a single pass and is more predictable.
  • However, changes to a child component are not pushed to the parent
    • Updating an input parameter on a child component will not update the corresponding value in the parent component
  • Changes from the child to parent use output bindings and event emitters, but this exists outside the change detection process
  • This is one of the reasons for the speed boost over Angular JS
    • Angular JS used bidirectional data flow, so you could update a parent through a child
    • This triggers multiple digest cycles to get the data model to stabilize
