Extending Angular Code

Inheritance

  • Since components are classes, you can extend a base class to share functionality between two components
  • This can lead to situations you do not necessarily want, such as:
    • Inheriting functionality in one component that you dont want/need just to get it into the other component
    • Good practice to only expose an API that you want the other side to use

Composition with Directives

  • Another approach is to deconstruct separate functionalities into their own directive
    • You would then decorate each host component in your template with only the directives that are needed
  • Downside is you have to explicitly add the desired directives each time you add the host components to any of templates
    • Leads to code duplication in templates

Directive Composition API

  • Requires Angular 15+
  • Directives can be added to a component class instead of templates
  • This uses the HostDirectives property in the Component decorator
    • This is an array of directives that are automatically applied to the host element of the component
    • Means you do not have not have to explicitly add these directives to the template when you add the component
  • One catch is inputs and outputs are components are treated as private by default
    • Add the proerties to the hostDirectives configuration to allow them to be used publically
    • See example for how to do this
    • You can also alias properties so you can use a different name in your templates
  • Any directives used must be standalone
