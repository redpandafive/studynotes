ViewChild

  • @ViewChild and @ViewChildren are used to get a reference to components that you define in the view template of your component
  • Use this when the parent component class must refer to the child component class
  • Allows you to inject the child component into the parent as a ViewChild
    • Import references to the ViewChild decorator and the AfterViewInit lifecycle hook
    • Inject using the @ViewChild decorator
    • You can now reference the child component through this variable, but it is not available until after Angular displays the parent view
    • Need to use the ngAfterViewInit hook to work with it. The child component is not available until after Angular displays the parent view
    • This can also be done with templates by referencing a template reference

Example

@ViewChild(CountdownTimerComponent) private timerComponent: CountdownTimerComponent
