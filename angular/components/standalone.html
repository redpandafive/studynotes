Standalone Components

  • Reduces the need for NgModules
  • Specify their dependencies directly instead of getting them through NgMoudles

Syntax

  • Standalone components have several additional properties on the Component decorator
    • standalone
      • Boolean value that indicates if this component is standalone
    • imports
      • Array of components, directives, pipes
      • This can also include Module names when you need items that are not standalone (such as ReactiveFormsModule, MatButtonModule)

Migration

  • Angular CLI includes a migration schematic to convert an existing application to standalone
  • Start with ng generate @angular/core:standalone
  • This includes three modes
    • convert-to-standalone: converts all your components to standalone components, except the ones declared in your main module.
    • prune-ng-modules: removes all the modules that arenâ€™t necessary anymore.
    • standalone-bootstrap: converts your main.ts to use the bootstrapApplication function
      • Where it can, it will use the appropriate provide-prefix function to import providers, such as provideRouter() for the RouterModule and provideHttpClient() for the HttpClientModule
  • A complete migration would require
    • Running each of three modes consecutively (testing builds between each step)
    • Cleaning up any remaining NgModules

Scenarios

  • You need to include a component that is not standalone
    • Include the Module name in the imports array
  • You need to import a standalone module into NgModule based applications
    • Include the component into the imports array of the existing NgModule decorator
