Basics

  • There are two methods of building forms in Angular:
    • Reactive forms
      • more robust, scalable, reusable, and testable
      • model driven approach to handing form inputs
    • Template driven forms
      • useful for adding a simple form to an app
  • Both approaches use the same underlying building blocks
    • FormControl
      • Tracks the value and validation status of an individual form control
    • FormGroup
      • Tracks the same values and status for a collection of form controls
    • FormArray
      • Tracks the same values and status for an array of form controls
    • ControlValueAccessor
      • Creates a bridge between Angular FormControl instances and native DOM elements
  • Both approaches use a form model to track value changes between Angular forms and form input elements

Select Controls

  • Three ways to get the current value of a select control
  • Method #1: Use the change event on the select control
    • Add a template variable to the select control
      <select #teams>
    • Bind the change event on the control to a method
      <select #teams (change)="onSelected(teams.value)">
    • Assign the value into a component property
      export class TeamWithChangeEventComponent {
      selectedTeam = '';
      onSelected(value:string): void {
      this.selectedTeam = value;
      }
      }
  • Method #2: Use ViewChild and a template reference variable
    • Add a template variable to the select control
      <select #teams>
    • Bind the change event on the control to a method, but do not pass any parameters
      <select #teams (change)="onSelected()">
    • Use a ViewChild decorator to reference the tempalte variable to access the current value
      export class TeamUsingViewChildComponent {
      @ViewChild('teams') teams!: ElementRef;
      selectedTeam = '';
      onSelected():void {
      this.selectedTeam = this.teams.nativeElement.value;
      }
      }
  • Method #3: Use NgModel and 2 way data binding
    • Use the NgModel directive in your Template
      <select [(ngModel)]="selectedTeam">
      <option default>Pick a team</option>
      <option>Lakers</option>
      <option>Miami</option>
      <option>Cleveland</option>
      </select>
    • This will bind to a property named selectedTeam on your component

Validation - Custom Validators

  • Custom validators can be implemented as classes
    • Use the Directive decorator on this class
      • This should contain a selector
    • The class name should end in a Validator suffix
    • The class should implement Validator class
    • Contains a single method, validate
      • Takes a FormGroup object as a parameter
      • Returns { [key: string]: any }
    • To access a control named address in the FormGroup
      • formGroup.controls['address']
    • You can navigate up the FormGroup hierarchy by accessing
      (<FormGroup>formGroup.root).controls["controlInParentFormGroup"]
    • Returning null means validation is passed
  • Custom validator classes must be added to Angular's validator 'service'
    • Import NG_VALIDATORS from @angular/forms - this is essentially a service
    • In the directive config object add
      Providers: [{provide: NG_VALIDATORS, useExisting: LocationValidator, multi: true }]
  • Without the multi: true parameter, this would break built in validators since LocationValidatior is being configured to be injected
  • The 3rd parameter adds the new validator to the collection of services provided by NG_VALIDATORS that are built in by Angular
