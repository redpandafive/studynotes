Reactive Forms

Basics

  • Requires more code, but allows more dynamic forms
  • Makes validation logic unit testable
  • Testing is easier since status and data are queried and manipulated through the control without interacting with the change detection cycle
  • Immutability
    • Keeps the data model pure by providing it as an immutable data structure
    • Each time a change is triggered, the FormControl instance returns a new data model rather than updating the existing data model
  • In Reactive forms, the form model is the source of trust (provides the value and status of the form element at a given point in time)
  • Built around observable streams, where form inputs and values are provided as streams of input values which can be accessed synchronously

Key Differences with Template Driven Forms

  • Template Driven
    • Setup of form model = implicit, created by directives
    • Data model = unstructured and mutable
    • Data flow = asynchronous
    • Form validation = directives
  • Reactive
    • Setup of form model = explicit, created in component class
    • Data model = structured and immutable
    • Data flow = synchronous
    • Form validation = functions

Basic Form with No FormGroup

Register the reactive forms module to include the reactive form directives needed
import { ReactiveFormsModule } from '@angular/forms';
Add a FormControl instance to the component
export class NameEditorComponent {
name = new FormControl('');
}
Register the FormControl in the template
<input id="name" type="text" [formControl]="name">

Form Using a FormGroup

  • There are two ways to grouping multiple related controls into a single input form
    • FormGroup
    • FormArray
Create a FormGroup instance on the component
  • Provide the constructor with an object of named keys mapped to their control
profileForm = new FormGroup({
firstName: new FormControl(''),
lastName: new FormControl(''),
});
Associate the FormGroup model and view.
<form [formGroup]="profileForm"> <label for="first-name">First Name: </label> <input id="first-name" type="text" formControlName="firstName"> <label for="last-name">Last Name: </label> <input id="last-name" type="text" formControlName="lastName"> </form>
The FormGroup directive listens for the submit event emitted by the form element and emits an ngSubmit event that you can bind to a callback function. Add an ngSubmit event listener to the form tag with the onSubmit() callback method.
<form [formGroup]="profileForm" (ngSubmit)="onSubmit()">
