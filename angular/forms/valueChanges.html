ValueChanges Events

  • An event raised whenever the value of one class that implements AbstractControl changes, including:
    • FormControl
    • FromGroup
    • FormArray
  • Allows us to track chagnes made to the value in real time and respond to them
    • Can validate a value
    • Calculcate a computed field
  • Returns an observable that you can subscribe to and get the latest value of the control
    • This gets the latest value of the control

Event Bubbling

  • ValueChanges events are fired immediately after new values are updated, but before the change is bubbled up to its parent
    • An event listener on a specific control will fire before an event listener on its containing form
  • ValueChanges event of a FormGroup or FormArray is fired when the value of any of its child control values changes
  • To prevent event bubbling, use the emitEvent option
    this.fg.get("firstname").setValue("", { emitEvent: false });

Setting Up

  • To subscribe to changes in a form, subscribe to the top level FormGroup
    this.fg.valueChanges.subscribe(x => {
    console.log(x);
    })
  • To subscribe to a specific control
    this.fg.get("firstname").valueChanges.subscribe(selectedValue => {
    // log the latest value of firstName
    console.log(selectedValue)
    // log out the latest value
    console.log(this.reactiveForm.get("firstname").value)
    })
