Directive Basics

  • Directives are classes that add additional behavior to elements
    • They change the appearance or behavior of an element, component, or another directive
    • When directives are rendered, Angular transforms the DOM according to the instructions given by the directive
  • Components are technically Directives
    • The main difference between a Directive and a Component is that a Directive does not have a template

Types

Attribute

  • Attribute directives change the appearance or behavior of an element, component, or another directive
    • Listen to and modify the behavior of other HTML elements, attributes, properties, and components
  • Usually applied to elements as if they were HTML attributes
  • You can add several attribute directives to the same host element

Structural

  • Responsible for HTML layout by shaping or reshaping the DOM's structure, typically by adding, removing, or manipulating elements
    • Alter the page layout by adding, removing, and replacing elements in DOM
  • Creates an embedded view from the Angular generated ng-template and inserts that view in a view container adjacent to the directive's original host element
  • These are limited to a single structural directive per host element
    • This is due to precedence and their complexity
    • For instance, cannot have *ngFor and *ngIf on the same host element
    • use ng-containers so you don't introduce extra levels of HTML
  • Apply to a host element - directive then does what it is supposed to do with that host element and its descendants
  • An * precedes the directive attribute name to identify it as a structural directive
    • No brackets or parentheses are needed
    • This is a convenience notation and the string that follows is a microsyntax rather than the usual template expression
    • This is de-sugared into a marked-up ng-template that surrounds the host element and its descendants
  • Common built-in directives are:
    • NgIf
    • NgFor
    • NgSwitch

Best Practices

  • Use attribute directives when you have presentation logic without a template
    • An element may have more than one attribute directive applied
  • Use @HostListener and @HostBinding decorators
    • Favor over host property of @Directive and @Component decorators
  • Use lower camel case for naming selectors of directives
  • Use a custom prefix
  • Don't use the ng prefix
