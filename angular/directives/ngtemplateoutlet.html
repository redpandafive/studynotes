NgTemplateOutlet

  • Structural directive to dynamically insert a template in various sections of our DOM
  • Provides more flexibilty than the ng-content
  • Since ngTemplates do not render themselves, this directive inserts the specified templated into the DOM does
    • The element containing the directive is "replaced" with the template contents
  • Useful scenario
    • Allows you to define a component that accepts an input containing data and an input containing a reference to a template
    • This allows you to pass in a template from outside the component to use
    • Good idea to supply a default template if one is not set
    • See this page for more details

Simple Example

  • Template is rendered in both the div and ngContainer tagged with the NgTemplateOutlet directive
  • The text within the NgContainer is not displayed
  • The div tag is not rendered
<ng-template #template1> This is a sample template </ng-template>
<ng-container *ngTemplateOutlet="template1">This text is not displayed</ng-container>
<div *ngTemplateOutlet="template1">
</div>

Passing Data

  • Data can be passed by settting the ngTemplateOutletContext input property of the NgTemplateOutlet element
    • This should be an object, such as:
      • An object in a componet
      • An inline object
  • Object keys will be available for binding by the local template let declarations
  • Using the key $implicit in the context object to set a default value for any let parameters that do not have a match in the context object

Syntax

  • The context input can be set within the NgTemplateOutput expression
    • In the template
      <ng-container *ngTemplateOutlet="greet"></ng-container>
      <ng-container *ngTemplateOutlet="eng; context: myContext"></ng-container>
      <ng-container *ngTemplateOutlet="svk; context: myContext"></ng-container>
      <ng-template #greet><span>Hello</span></ng-template>
      <ng-template #eng let-name>
      <span>Hello {{name}}!</span>
      </ng-template>
      <ng-template #svk let-person="localSk">
      <span>Ahoj {{person}}!</span>
      </ng-template>
    • In the component
      myContext = {$implicit: 'World', localSk: 'Svet'};
  • The context input can also be set seperately on the element
    • In the template
      <ng-container [ngTemplateOutlet]="optionTemplate" [ngTemplateOutletContext="{ prop1: val1, prop2: val2}"> ...
  • This is a shorthand version (desugars into the form below)
    <ng-container *ngTemplateOutlet="tierTemplate; context: { $implicit: reason, idx: 5 }"></ng-container>
  • This is an alternative longer format. It is equivalent to the above.
    <ng-container [ngTemplateOutlet]="tierTemplate" [ngTemplateOutletContext]="{ $implicit: reason, idx: 5 }"></ng-container>
