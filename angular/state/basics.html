State Basics

Terminology

  • State is information that is tracked by the application. This includes:
    • View State = information about the view, such as if a loading icon should be shown
    • User Information = user name and roles used to tailor the application to the user
    • Entity data = data displayed and manipulated, such as product data
    • User selection and input
  • Component State
    • Data stored on components, often as class properties
    • Could be things like current sort order or selected item
  • Service State
    • Data retained throughout the application

Example Using Signals

  • https://github.com/DeborahK/Angular-Signals
  • This example uses a service (CartService) to represent a shopping card
  • Includes a signal that holds items (an array of CartItem class) that are currently in the shopping cart
  • Includes computed signals for subtotal, delivery fee, tax, and total price
    • Each of these is a computed signal
    • They reference the CartItem signal as well as each other to automatically update their values the CartItem signal changes
  • CRUD methods to add, remove and update items in the CartItem signals. Prices will update since they respond to changes in the signal.
