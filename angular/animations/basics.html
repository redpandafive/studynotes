Animations

  • Built on CSS functionality, which means you can animate any property that the browser considers animatable

Setup

Add provideAnimations to the providers list
bootstrapApplication(AppComponent, {
providers: [
provideAnimations(),
]
});
Must import the animation capabilities into root Angular application modules (or component, if using standalone components)
import { BrowserAnimationsModule } from '@angular/platform-browser/animations';
Import functions into components that will use specific animation functions
import { trigger, state, style, animate, transition } from '@angular/animations';
In the component file, add a metadata property called animations within the component decorator
@Component({
selector: 'app-root',
templateUrl: 'app.component.html',
styleUrls: ['app.component.css'],
animations: [
// animation triggers go here
]
})

State Function

  • Use to define different states to call at the end of each transition
  • This function takes two arguments
    • A unique string used to name the state (such as 'open' or 'closed')
    • A style function
      • Defines a set of styles to associate with the name

Style Function

  • Declares a key/value object containing CSS properties / styles
  • Associates them with an animation state
  • Multiple style attributes can be set at the same time
  • Unlike CSS, style properties should use camel casing for styles that contain dashes (or quotes)
    • Alternatively, you can wrap them in quotes, such as 'background-color'
Two sets of CSS style attributes are defined, one for the 'open' state and one for the 'closed' state
state('open', style({
height: '200px',
opacity: 1,
backgroundColor: 'yellow'
})),
state('closed', style({
height: '100px',
opacity: 0.8,
backgroundColor: 'blue'
}))

Transition Function

  • Specifies the changes that occur between one state and over a period of time
  • Use to define the length, delay, and easing of a transition
  • Accepts two arguments
    • An expression that defines the direction between two transition states
    • An array of animate() steps
      • Use to define keyframes() function for multi-step animations

Animate Function

  • Accepts the timing and styles input parameters
  • Timing parameters can either be
    • A number (representing milliseconds) representing duration
    • A string containing up to three items
      • duration - in milliseconds or seconds
        • Required
        • Use '100ms' for milliseconds and '.1s' for seconds
      • delay - in milliseconds or seconds
        • Uses same syntax as duration
      • easing - how the animation accelerates and decelerates during its runtime
          • ease ease-in ease-out ease-in-out cubic-bezier() function cal
        • See this for easing functions
  • Examples
    • Run for 200ms, with no delay. Use a standard curve to start slow, accelerate in the middle, and then decelerate slowly at the end: '0.2s ease-in-out'
    • Start immediately, run for 200ms. Use an acceleration curve to start slow and end at full velocity: '0.2s ease-in'
    • Wait for 100ms, run for 200ms. Use a deceleration curve to start out fast and slowly decelerate to a resting point: '0.2s 100ms ease-out'
Transitions are defined to specify changes that occur between one state and another over a period of time
  • The transition function accepts two arguments
    • An expression that defines the direction between two transition states
    • One or a series of animate() steps
    • The animate() function
      • Defines the length, delay, and easing of a transition
        • This is the first parameter, the timing parameter
      • Designates the style function for defining styles while transitions are taking place
        • This is the second parameter, the Styles input paraemter
