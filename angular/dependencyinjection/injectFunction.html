Injector Function

  • Added in Angular 14
  • This provides an alternative syntax to injecting dependencies
  • Old way: use constructor injection
    constructor(private route: ActivatedRoute) {}
  • New way: call inject method and supply an Injector Token
    const route = inject(ActivatedRoute);
  • Benefits
    • Cleaner syntax
    • Type inference
    • The injected dependency can provide default values for other properties (in a way that is ECMAScript compliant)
  • Limitations
    • Cannot be used inside of an event hander or method, only at constructor time
  • Use Cases
    • Simplifies inheritance setups
      • Without it, you are required to pass all dependencies through all constructors in the inheritance chain using the super method
      • With it, you are only required to call inject within the component where you need the dependency
