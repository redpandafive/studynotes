Reactive Forms - FormControl

  • Extension of the AbstractControl class
  • Corresponds to an individual control inside of an HTML form
  • This is the basic building block class that represents a single control on the form
    • In Reactive forms, this is one of the main building blocks you work with directly
    • These are used in Template Driven forms, but they are under the hood
  • Tracks the validity state and value of a group of FormControl instances
  • This is intended for use cases where keys are know ahead of time
    • If you need to dynamically add and remove controls, use FormRecord instead
  • You will define one instance for each of your form elements
  • You can access the value of a FormControl object using the following
    • The value property - gives a snapshot of the current value
    • Using the valueChanges observable
      • can listen for changes in the:
        • template using the AsyncPipe
        • component using the subscribe method
    • Example just uses interpolation to display the value, which is updated on each change, so not really sure there the above come from

Constructor

  • Takes three parameters:
    • Initial data value, including null
    • Optional array of synchronous validators
    • Optional array of asynchronous validators
