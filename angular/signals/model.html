Model Signals

  • Allows Signal based two way data binding
  • Uses an underlying ModelSignal class
  • Not a replacement for the NgModel directive, but they have overlapping concepts
    • ngModel is syntax sugar for three items:
      • A ngModel directive on the element (or your own component if you are implementing your own data binding)
      • An Input to the ngModel directive (or an Input defined on your component)
      • An Output event handler to handle an event from the directive (or an Output defined on your component)
    • The model function would take the place of the Input and Output definition from the custom component
The following code ...
export class TwoWayBindingComponent {
&Input() expand = false;
&Output() expandChange = new EventEmitter<boolean>();
...
}
Can be replaced with ...
...
expand = model<boolean>();
...
You would use this in the TwoWayBindingComponent template using the following tentative syntax:
<button (click)="expand.set(!expand())">
{{ expand() ? 'Les Info' : 'More Info' }}
</button>
Outside of the TwoWayBindingComponent template, you would bind using the following syntax to bind to the showMore component property:
<app-banner [(expand)]="showMore">
Alternatively, you can bind to a signal
export class AppComponent {}
showMore = signal(false);
....

Required

  • You can make a model value required using the following syntax: model.required<boolean>
  • This functions like the required property that can be used with Input
