Dependency Injection (DI) vs Inversion of Control (IoC)

  • DI is the act of supplying a low level or dependency class via a constructor, method, or property
    • Dependent classes can be inverted to interfaces or abstract class, leading to loosely coupled systems
  • IoC
    • A system's flow of control is inverted compared to procedural programming
    • Example is an IoC container, whose purpose is ot inject services into client code without having the client code specifying the concrete implementation
    • The control in this instance that is being inverted is the act of the client obtaining the service

Design Patterns

Decorator Pattern

  • Allows behavior to be added to an individual object (either statically or dynamically) without affecting the behavior of other objects from the same class
