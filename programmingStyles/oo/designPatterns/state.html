State Pattern

  • Used to represent the state of an object separate from all other behavior
  • Allows an object to alter its behavior when its internal state changes
    • This is done by swapping internal state objects that implement state dependent behavior
    • An object defers all state based behavior to a dependent state subclass
  • Eliminates the need for a mass of case statements within methods on the object
