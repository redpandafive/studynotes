Imperative

  • Imperative code is step by step instructions to the compiler about how to perform a task
  • It explains exactly how it needs to occur
  • You usually have to look in other parts of the code to get a full picture about what a chunk of code actually does
    • A variable can have a value returned by a service.
    • If it is modified later on in the code, you have to have a wider view of the code to see the role of the variable.
    • This is where immutable data can be a benefit.

Declarative

  • Code that describes what you need to do, not necessarily how to do it
  • Example language is SQL - you tell the database WHAT you want, not HOW to fetch it
  • Most declarative solutions are an abstraction over some imperative impelmeation

Choose the odd numbers from a collection in C#
Imperative
  • Create a result collection
  • Step through each number in the collection
  • Check the number, if it's odd, add it to the results
List<int> results = new List<int>();
foreach(var num in collection)
{
if (num % 2 != 0)
{
results.Add(num);
}
}
Declarative
var results = collection.Where( num => num % 2 != 0);

Reactive

  • A type of declarative programming paradign focused on
    • Useage of data streams
    • Change propagation
  • Idea that everything is a stream
    • A sequence of data elements made available over time
    • Think of as items on a conveyor belt being processed one at a time reather than in large batches
    • Data and events can be represented as streams of data over time
      • Your program reacts to changes in these streams
    • When something changes upstream, everything downstream updates as well
  • Examples
    • Registering an event listener on a DOM element
    • RxJS
