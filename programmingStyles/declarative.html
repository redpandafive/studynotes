Declarative

  • Code that describes what you need to do, not necessarily how to do it
    • Example language is SQL - you tell the database WHAT you want, not HOW to fetch it
  • Usually uses some reactive mechanism (such as a RxJS observable or a Signal)
    • This way the variable that holds the mechanism never changes after its declaration, but the value can change over time
  • Additional values can be derived from a reactive mechanism, so they are automatically updated when the source updates
    • Example is Computed from Signals
  • Vs Imperative code style
    • Imperative style can change a value at multiple points in the code
    • Tracing all the changes make to the value requires tracing through the codebase to find each point the value was changed
    • The definition of a "thing" requires looking at multiple parts of the code
  • Rules to get your in a Declarative style programming mindset:
    • Make sure things never change
      • This could be creating a property on a component and changing its value at multiple points on the application
      • Don't reassign a value to a variable after declaring it
        • Do this by using some reactive mechanism like a RxJS observable or Signals
        • The assignment never changes, but the value can change over time using the reactive mechanism
      • Example would be
        • Assigning an empty array as a property on the component
        • Calling an async service to fetch a value from the NgOnInit method
        • When the observable completes, assign that value to the component property (changing its value)
    • Always make sure to speak to the manager
      • Changes to values should be done at the start of the data flow, then everything downstream will react to the change
  • Most declarative solutions are an abstraction over some imperative implementation

Imperative vs Declarative Examples

  • Declarative code's value (versus Imperative coding) increases as complexity increases
A non-programming example would be "How do I get to your house from downtown?" (from https://ui.dev/imperative-vs-declarative-programming)
Imperative (explicit step by step instructions for how to do it)
  • Go out of the north exit of the parking lot and take a left.
  • Get on I-15 going North until you get to the 12th street exit.

Declarative (gives the end result, but not how to do it)
  • My address is 298 West Immutable Alley, Eden, Utah 84310
Choose the odd numbers from a collection in C#
Imperative
  • Create a result collection
  • Step through each number in the collection
  • Check the number, if it's odd, add it to the results
List<int> results = new List<int>();
foreach(var num in collection)
{
if (num % 2 != 0)
{
results.Add(num);
}
}
Declarative
var results = collection.Where( num => num % 2 != 0);
