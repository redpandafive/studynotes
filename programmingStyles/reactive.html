Reactive

  • A type of declarative programming paradigm focused on
    • Usage of data streams
    • Change propagation
  • Code is reactive when an input change leads to an automatic change in output
    • Changes are emitted and the application reacts to those changes appropriately
    • Code can react to
      • User actions
      • State changes (instead of procedurally calling methods)
  • Data streams can be combined to compose data from multiple sources
  • vs Imperative programming
    • In imperative programming, variables are assigned whent an expression is first evaluated
    • In reactive programming, variables observe and react to changes
  • Idea that everything is a stream
    • A sequence of data elements made available over time
    • Think of it as items on a conveyor belt being processed one at a time reather than in large batches
    • Data and events can be represented as streams of data over time
      • Your program reacts to changes in these streams
    • When something changes upstream, everything downstream updates as well
  • Examples
    • Registering an event listener on a DOM element
    • RxJS
