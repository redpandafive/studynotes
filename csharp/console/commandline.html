System.CommandLine

  • This replaces the deprecated Microsoft.Extensions.CommandLineUtils

Terminology

  • Arguments
  • Commands
    • Specifies an action or defines a group of releated actions
  • Options
  • Tokens
    • Strings delimited by spaces
    • Command line input is parsed into a set of tokens, which are strings delimited by spaces
  • Verbs
    • Also known as subcommands

Command Line Input Parsing

  • Input is parsed into tokens
    • Tokens can contain spaces if enclosed by quotation marks
  • Tokens are interpretted as
    • Commands
    • Options
    • Arguments
  • The app being invoked determines how the tokens after the first one are interpreted

Commands

  • Specifies an action or defines a group of releated actions
  • Examples
    • dotnet run = run is a command that specifies an action
    • dotnet tool install = install is a command that specifies an action, and tool is a command that specifies a group of related commands.
  • Root command
    • Specifies the name of the app's executable
    • Examples
      • dotnet tool install - donet is the root command
  • Subcommands
    • Also called Verbs
    • These can have their own subcommands
    • dotnet tool install = tool is a subcommand and install is a subcommand of tool

Options

  • A named parameter that can be passed to a command

Simple Build

  • To build a command line project, use the following command: dotnet build
