Expression Body Members

  • Allow you to replace statements blocks with expressions (just like with lambda expressions)
  • Looks like a lambda expression (it uses the token =<), but it does not mean there is a delegate involved
  • Purely syntax sugar, so no overhead is incurrent
Example
public Point Move(int dx, int dy) => new Point(x + dx, y + dy);)"
To make a property read only
public string Name { get; }"

To make a getter using an expression body member
public string Name => _name"
To make a getter and setter using an expression body member
public string Name { get => _name; // lambda expression – returns a value set => _name = value; // lambda statement – no return value } "
To make a getter and setter with a exception throw
public string Name { get => _name; set => _name = value ?? throw new InvalidOperationException(); }"
These can be declared on a void method, but they use must use a statement expression
