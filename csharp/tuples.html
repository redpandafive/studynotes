Tuple Types and Tuple Literals

  • Added to C# 7 to provide a lightweight way to return multiple values from a single function
  • Advantages
    • Can contain different types
    • Good alternative to out parameters
    • Not immutable
    • Cleaner syntax
    • Value types
  • Legacy Tuple class
    • Passed by reference, so stored on heap
    • Must access members using Item1, Item2, etc...
  • New Tuple version advantages
    • Value types and value equality; two tuples are equal if all their elements are pairwise equal
    • These are mutable (unlike Tuple class)
    • Simpler than using the Tuple object
    (int value1, int value2) GetPrice(int itemId) { ... var product = (500, 500); return product;"
  • Requires an additional NuGet package to work in certain versions of C# - System.ValueTuple
  • To access the values within a tuple, you must follow a convention
    • var price = (500, 100) assigned to a variable price would be accessed by price.Item1 and price.item2
    • var price = (a: 500, b: 100) would use price.a and price.b
    • if returning from a function, you can assign friendly names in the method signature
      (int price, int discount) GetPrice(int itemId) { ..."
  • Can also be used as keys to a dictionary
  • Return type = (String firstname, String lastName)
    • Can have it without the parameter names, but then you must refer to the individual items as Item1 and Item2 (to be compatible with previous version)
    • Don't have to have the same names are the variables that hold the values you are returning
  • Can define the receiving parameters:
    • var name =methodThatReturnsTuple()
    • can use name.firstName and name.lastName to get values
    • Can deconstruct the Tuple into individual variables

Tuple Deconstruction

  • Allows you to access a tuple member directly with its name without using any tuple variable name
  • Three ways to do this
    • for function GetPrice that returns a tuple (int price, int discount)
      (int price, int discount) = GetPrice(); var (price, discount) = GetPrice(); var (int price, int discount) = GetPrice();"
    • This can be used when the variables have already been declared
      (price, discount) = GetPrice();"
    • Use when discount is declared, but price is not (must use C# 10)
      (price, int discount) = GetPrice();"
    • Can now access variables price and discount
