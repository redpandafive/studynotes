Debugging and Exception Handling

  • Commands

Debugging

  • Run to click
    • If you are stopped at a breakpoint, you can hover of just to the left of any line of code
    • Pressing the green arrow will run to that point
  • Run to cursor
    • Right click to execute to the cursor position
  • Null reference analysis
    • Popup for NullReferenceExceptions will display the variable name that caused the exception

Exceptions

  • Are thrown at different times depending on the method type
  • Async
    • These are observed when the task has completed
  • Method iterators
    • Exceptions are surfaced only when the return sequence is enumerated
    • This can be alleviated by putting iterator logic into a local function
