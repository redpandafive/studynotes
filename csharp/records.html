Records

  • Commands

  • Reference type that is immutable
    • Intended to support immutable data models
  • Features
    • Concise syntax for creating a reference type with immutable properties
    • Value-based equality
    • Built in formatting for display
    • Concise syntax for nondestructive mutations
    • Support for inheritance hierarchies
  • The compiler auto generates several things out of the box of you choose that route
    • A public auto implemented property for each positional parameter provided in the records declaration
      • Declared using get and init keywords
    • A primary constructor whose parameters match the positional parameters on the records declaration
    • A Deconstruct method with an out parameter for each position parameter
    • A ToString method
  • Positional parameter declaration makes an object immutable
    • Declares properties with the init keyword instead of set
    • Useful when working with multi-threaded environments
    • Shallow immutability
      • Cannot change value types
      • Cannot change the references that reference types point to
      • Can change the data on reference types
      • Cant change an array, but you can modify fields in the objects in the array
  • If you choose to do the declaration yourself, you can
    • Mix positional parameters with explicitly declared one
    • No position parameters at all
    • Modify properties to be mutable
  • Deconstructor method is automatically created on records with two or more positional parameters
  • Starting with C# 10, you can define records structs
    • Public readonly record struct ...
      • immutable
    • Public records struct ...
      • Mutable

Value Equality

  • Unlike classes, with records, two object are equal if they are of the same type and store the same values
  • Comparisons between two records is done by comparing property values (not reference)
  • The complier will automatically synthesize
    • An override of the Object.Equals(Object) method
    • A virtual Equals method whose parameter is the records type
    • Override of Object.GetHashCode()
    • Overrides of operators == and !=
  • You can write your own versions, in which case the compiler doesnâ€™t try to implement anything you have provided

Nondestructive Mutation

  • Creates a copy of an instance with modifications
  • Uses the with keyword
    • Combine with an object initializer to modify the values to be changed
  • Non positional properties must have an init or set accessor to be changed by a with expression
  • Example - given a record person1 with parameters string FirstName and string LastName
    code='Person person2 = person1 with { FirstName = "NewFirstName" }'

Build In Formatting for Display

  • The compiler will automatically generate an override to the Object.ToString method
  • You can provide your own for customized formatting

Inheritance

  • Records can inherit from other records
  • Similar syntax to specifying inheritance for a class
