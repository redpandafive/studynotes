Generics

  • Commands
  • Allows you to create multiple type safe and efficient objects using a single class definition
    • Type definition is deferred until the class is declared and instantiated by the client
    • Cannot be used as is - more like a blueprint for a type
    • Client code must declare and instantiate a constructed type by specifying a type parameter inside the angle brackets
  • Type parameter that is a placeholder for a specific type that a client specifies when they create an instance of the generic type
    • Type can be any recognized by the compiler
    • These can be used as
      • Placeholder for the types of object that is stores
      • Types of its fields
      • Parameter types of its methods
      • Type of a return value

.NET namespaces

  • System.Collections.Generic
    • Generic collection types
    • List, Dictionary, etc.
  • System.Collections.ObjectModel
    • Additional generic collection types
    • ReadOnlyCollections

Terminology

  • Generic type definition
    • Class, structure, interface that functions as template
    • Cannot create instances that are generic type definition
  • Generic type parameters
    • Placeholders in a generic type or method definition
  • Constructed generic type
    • Result of specifying types for the generic type parameters of a generic type definition
  • Generic type argument
    • Type that is substituted for a generic type parameter
  • Generic type
    • Includes both constructed types and generic type definitions
  • Variance
    • Covariance and contravariance
    • Allow you to construct generic types whose type arguments are more derived or less derived than a target constructed type
  • Constraints
    • Limited placed on generic type parameters
  • Generic method definition a method with two parameter lists
    • A list of generic type parameters
    • List of formal parameters

Benefits

  • Type safety
  • Reusability and less code
  • Better performance since boxing value types is not needed

Naming Recommendations

  • Name generic type parameters with descriptive names, unless a single letter name is completely self-explanatory and a description name would not add value
  • Use T as the type parameter name for types with one single letter type parameter
  • Prefix descriptive type parameter names with T

Recommendations

  • Apply the maximum constraints possible that will still let you handle the types you must handle

Generic Classes

  • Generic class can inherit from
    • Concrete type classes = no generic parameters
    • Closed constructed type = Generic base class with the generic parameter set to a specific type
      Class NodeOpen<T> : BaseGeneric<int> {}"
    • Open constructed type = generic base class with a generic parameter
      Class NodeOpen<T> : BaseGeneric<T> {}"
  • Generic classes must supply type arguments for any base class type
    • This causes a compiler error
      Class NodeOpen<T> : BaseGeneric<T, U> {}"
    • Generic classes that inherit from base open constructed types must specify constrains that are a superset of the constraints on the base class
    • Concrete classes can inherit from closed constructed base classes
      • Cannot inherit form generics with open constructed type
      • No way to supply the type argument to instantiate the base class

Generic Methods

  • Can appear on generic or non-generic class types
  • A method is not generic because it belongs to a generic type or it has formal parameters whose types are generic parameters of the enclosing type
  • It is generic only if it has its own list of type parameters
    • T Method1(T arg) ...
    • It is generic even if its enclosing class is not generic
    • Belonging to a generic class
  • Definition
    T DoStuff<T>(T arg) { T temp = arg; Return temp; } "
  • Can call this as
    var DoStuff<int>(5); var DoStuff(5); "
  • If defined in a generic class, do not need the >T< after the method name if you are using the same identifier
    • If it is a different one, then you will need it

Constraints on Type Parameters

  • You can tell the compiler about the capabilities a type argument must have
    • By default, it can be any type (System.object)
    • Allows you use the operations and method calls of the constraining type
  • Example
    • “Employee” can be
    • “Struct”, “class”, “notnull”
    • a base class name
    • an interface name
    • new() = argument must have a public parameterless constructor
    • for more, see https://docs.microsoft.com/en-us/dotnet/csharp/programming-guide/generics/constraints-on-type-parameters
    public class GenericClass<T> where T : Employee ..."
  • Multiple constraints can be applied using a comma separated list
    class Test<Y, U> where U : stuct, where T : Base, new()"
