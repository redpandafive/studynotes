Terminology

  • Commands
Arguments
  • Concrete values passed by code when it calls a method
  • Must be compatible with the parameter type
Assignment compatibility
  • An object of a more derived type can be assigned to an object of a less or equivalent derived type
Dereferencing
  • Access one of an object's members using the dot (.) operator
  • Example would be message.Length where message is a string object
  • Derefenceing a variable whose value is null triggers a System.NullReferenceException
Event Handlers
  • Nothing more than methods that are invoked through delegates
Expression
  • A combination of values and functions that are combined to create a new value
  • Can be used in places where a value is expected
  • In functional languages, everything is an expression, including statements
  • Multiple expressions can be composed together into larger expressions
  • Idempotent, calling the came expression for the same input any number of times always produces the same result
Expression-bodied Members
  • Define a members implementation in a very concise, readable form
  • Use whenever the logic for any supported member (such as method or property) consists of a single expression
  • General form is member => expression
  • Can be used on
    • Methods
      • Returns a value that matches the method’s return type or returns void for actions
      • Never uses the return keyword
    • Read-only properties
    • Properties
    • Constructors
    • Consists of a single assignment expression or a method call that handles constructor’s arguments
    • Finalizers
    • Indexers
Parameters
  • Defined on the method signature
  • Specifying the name and types
Signature
  • Signature is comprised of
    • Declaration such as class, struct, or interface
    • Access level such as private or public
    • Optional modifiers such as abstract or sealed
    • The return value
    • Method name
    • Method parameters
Statement
  • Just a standalone unit of execution and doesn’t return anything
  • Imperative languages rely heavily on statements to accomplish everything
  • Modify variables and change state
  • Do not guarantee idempotency
  • Must be evaluated in order

Keywords

  • this
    • Refers to current instance of the class
    • Also used as a modifier of the first parameter of an extension method
  • virtual
    • Allows the member to be overridden in a derived class, changing the implementation
    • Can be applied to a method, property, indexer or event declaration

Shallow immutability

  • Cannot change value types
  • Cannot change the references that reference types point to
  • Can change the data on reference types
  • Can't change an array, but you can modify fields in the objects in the array

Special Characters

  • @
    • Enables C# keywords to be used as identifiers.
      • The @ character prefixes a code element that the compiler is to interpret as an identifier rather than a C# keyword.
    • Indicates that a string literal is to be interpreted as a verbatim string literal
      • The following are interpreted literally
        • Simple escape sequences (such as "\\" for a backslash)
        • Hexadecimal escape sequences (such as "\x0041" for an uppercase A)
        • Unicode escape sequences (such as "\u0041" for an uppercase A) are interpreted literally
      • The following are not
        • Quote escape sequence ("") is not interpreted literally; it produces one double quotation mark.
        • Interpolated string brace escape sequences produce single brace characters
  • $
    • When prepended to a string, this indicates the string uses interpolation
