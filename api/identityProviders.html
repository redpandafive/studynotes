Identity Providers

  • Identify Providers has the responsibility to authentiate users and, if needed, safely provide proof of identity to an application
  • Handle identity and acces management related tasks, such as:
    • User registration and management
    • Locking out users
    • Password policies, strengths and resets
  • Better to handle this in one cental location and resuse them accross applications
    • Use accounts can be reused across applications
    • Users will no longer prove who they are in the application itself, but to the Identity Provider instead
  • Also known as
    • Authentication servers
    • Security token services
  • Examples include
    • Duende Identity server
    • Azure AD
    • Ping
    • Auth0

Duende IdentiyServer

  • Open ID Connect and OAuth 2 framework for ASP.NET Core
  • Developed by Duende Software
    • There is a paid version
    • There is a community license that allows for free usage under certain conditions
  • Completely open source
  • Certified by the Open ID Foundation
  • Part of .NET Foundation
    • Even used in some of the Visual Studio starter templates, such as Angular or Blazor
