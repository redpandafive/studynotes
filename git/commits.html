Reversing Commits with Reset

  • Allows you to rollback a commit by moving it back into working directiory or staging area
  • This can be used to throw changes away if needed
  • Best practice to not reset a commit to remote origin once you have updated it from your local repository

Commands

  • get add .
    • Sends specified modifications to staging
      • Use to add untracked file or files to the index so they will be tracked by Git
      • Also use to add a changed file that is already being tracked
    • Includes all modifications, deletions, and additions for all files in the current directory and all subdirectories
    • Issuing this from the root is equivalent to git add -all
    • It will not include any changes from parent directories
    • Once added, a file will show up as a change to be commmitted under the git status command output
    • Think of this as changes are staged
      • If you make changes to a file, stage the file, then modify the same file again, only the first change will be included in the commit
      • The file will show up under both "Changes to be commited" and "Changes not staged for commit"
      • Run the add command again on the file to include both sets of changes in the next commmit
  • git add -A and git add -all
    • Includes all modifications, deletions, and additions
    • Does not matter which directory you are in
  • git add -u
    • Stages modifications and deletions, but no new files
    • Use git restore to undo this command
  • git add *
    • Do not use this, not equivalent to above commands
    • The * is a shell command, not something Git specifically knows how to use
    • This will bypass .gitignore
  • git commit
    • Sends files from the index to the repository to create a snapshot
    • Without the -m option, issuing this command will open a text editor so you can write out a commit message
      • Default editor is Vi
      • Use :wq to save and quit
    • git commit -m "start of changes for quickfix"
      • Sends the files to the repository with an attached message
      • Commits staged changes to the repository
      • This will include a message of "start of changes for quickfix"
    • git commit -a -m "Some message"
      • Bypasses the staging step and commits changes directly from working area along with a message
  • git reset --soft xxxxx
    • Moves commit back into staging area
    • Useful if you want to regroup changes differently or add more changes before commiting
    • Moves head back to commit id xxxxx and undoes all commits beween that commit (excluding specified) and the current one (including current one)
  • git reset --mixed xxxxx and git reset --mixed xxxxx
    • Moves commit back into working directory
    • This is the default if nothing else is specified
    • Moves head back to commit id xxxxx and undoes all commits beween that commit (excluding specified) and the current one (including current one)
    • Reset current HEAD ot the specified state
  • git reset --hard xxxxx
    • Moves changes to the trash
    • This will also remove changes to working directory and staging, so be careful
    • Moves head back to commit id xxxxx and undoes all commits beween that commit (excluding specified) and the current one (including current one)
