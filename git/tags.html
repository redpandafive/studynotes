Tags

  • Simple Tags
  • Annotated Tags
  • Commands
  • Think of as a label for a commit
  • There are two types
    • Simple - points to a commit
    • Annotated - points to a tag object that points to a commit
  • Tags give you an additional way to checkout a commit
    • git checkout release_1 will checkout the commit with tag release_1
    • You can only use the checkout command with tags, not switch

Simple Tags

  • Also called lightweight tags
    • Tags that consist of just a tag name and no additional metadata
    • Very similar to branches, only they do not move
  • Stored in the .git/refs/tags subdirectory
  • Files are named after the tag name
  • Inside this file is a hash that references a commit hash ID
  • Examples
    • git tag release_1
      • Adds a simple tag of release_1 to the current commit

Annotated Tags

  • Contains additional metadata, such as a message
  • Add a -a switch to the command
  • Stored in the .git/refs/tags subdirectory
  • Files are named after the tag name
  • Inside this file is a hash, which points to a tag object
  • The tag object contains all the information in the tag, including
    • The tag
    • The user who submitted the tag
    • The date
    • The message, if included
    • A pointer hash that references its attached commit
  • Examples
    • git tag release_1 -a =, "First release, still unstable"
      • Adds a simple tag of release_1 to the current commit

Commands

  • git checkout tag_name
    • Checks out a commit with the tag name tag_name
    • Cannot use with the switch command
  • git tag
    • Displays the tags for the current release
  • git tag release -a "Message attached to tag"
    • Adds an annotated tag
