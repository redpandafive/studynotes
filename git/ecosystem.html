Using a Git Repository

  • Command Line
    • Git Bash is a good shell for this
  • Development Tool integration
    • Most modern development tools include Git integration, such as Visual Studio, Eclipse, Visual Studio Code
  • External Programs
    • Sourcetree is an example of a GUI for GIT so you do not have to use command line
  • Git based services
    • There are also Git based services, such as GitHub, GitLabs, and BitBucket

GitHub

  • A hosting service that will store Git repositories
  • Repositories are public by default, but you can do private repositories
  • Allows you to clone the hosted repository to your workstation
  • Also includes project management tools, such as issue tracking
  • Can be though of as a social network centered on coding
  • Has a preferredworkflow model (Pluralsight author calls this a Open Source workflow)
    • Projects are identified by combining the owner user name with the project name (ex. pluralsight/marathonfish)
    • Cloning
      • Creates a copy of the repository on your local machine
    • Fork
      • Creates a clone of a GitHub project into your GitHub account
      • You would download from this version to your workstation
    • If you want to push your changes to the original project, you must submit changes to the maintainer of the original project
      • You can push your changes to your GitHub account (the forked repository)
    • Pull request
      • You send a message to the original project maintainer notifying them about your updates.
      • After discussing the changes, the project maintainer can pull the changes to the original repository
