Starting a Working Area

  • git init
    • Creates a Git Repository on your computer
    • Adds the .git subdirectory in the targetted folder
    • Contains all the metadata for the project
  • git clone
    • Clone a repository into a new directiory

Work on the Current Change Area

  • git add
  • git mv
    • Move or rename a file, a directory, or a symlink
    • First parameter is the file to move or rename
    • 2nd parameter is the new filename
    • The file will show up with a status of renamed under git status
  • git reset
  • git restore
    • use to discard changes in working directory
    • git restore myFileName --staged will remove a file named myFileName from staging
  • git rm
    • Remove a file from the working tree and from the index
    • This stages the deletion and Git will stop tracking it
    • The file will be removed from the working directory
    • git rm --cached target-filename
      • Git will stop tracking this file and remove it from the index
      • This option WILL NOT delete the file from the working directory. It will show up as untracked under git status

Examine the History and State

  • git show 100644
    • Shows the contents of the object with the hash identifer that starts with 100644
  • git log
    • Shows the commit history for the repository
    • Displays the identifier, author, and date of the commit
  • git status
    • First line is the current branch
    • Next lines will display if there are changes to be aware of, including
      • Files in the project area that are untracked
      • Files that have been modified since the last commit
      • Files that are staged for the next commit - this will flag the file as a new file or a change to an existing file
      • If no files are staged or have outstanding changes, this will be empty
  • git status -s or git status --short
    • Shorter, more compact and concise version of git status
    • Three columns are displayed for each file
      • First column indicates the staging status of the file (green text)
      • Second column indicates the file has been modified, but remains in the working directory (red text)
      • Third column is the filename
    • Letters are used to indicate status
      • M = modified
      • A = new file added to staging area
      • ?? = new file untracked by Git
    • Examples
      • File has been staged as a modified change, file has changes that were made after it was staged
        $ git status --short M M File2"
      • Testfile9 is modified but not staged. Testfile22 is in the project, but not tracked. Testfile100 is a new file that has been staged. All others are modified and staged
        code='$ git status --short M "Test Folder/testfile10.txt" M "Test Folder/testfile6.txt" M "Test Folder/testfile9.txt" M testfile1.txt A testfile100.txt M testfile9.txt ?? testfile22.txt '

Grow, Mark, and Tweak Your Common History

  • git commit
  • git diff
    • Displays the differences between two snapshots
      • Dont need to include the entire hex code, just the first few characters
  • git log
    • Shows history of commits
    • These are displayed with most recent entries at the top
    • This command has lots of options that can be used
    • For each commit, it shows
      • Full SHA1 checksum identifier for the commit
      • Author
      • Commit datestamp
      • Commit message
    • git log -10
      • Only shows the 10 newest commits
    • git log --oneline
      • Gives a shortened summary of each commit
      • Each line represents a commit with a few digits of the SHA1 has identifier and the commit message
    • git log --stat
      • Gives a detailed view for each commit
      • Includes all the information from git log
      • Also includes the individual files that changed in the commit
      • Press q to quit
    • git log --patch
      • Displayes the full diff for each commit
      • Press q to quit
  • git checkout
