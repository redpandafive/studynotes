General Version Control Information

  • Software designed to manage changes made to files over time - gives long running history of a software project
  • Gives you infinite undo/redo functionality
    • Files / sets of files can be reverted to any previous versions
  • Not a backup system - you must backup and secure the repository
  • Allows you to juggle multiple versions of the same app at the same time
    • Example is a free and premium version of an app
    • You could have multiple branches of the same app
  • Makes sharing code among a team of multiple developers easier
    • Developers can share the same repo so painful integration phases lessened
  • Coordinating teamwork
    • The repository can be the source of truth for all users, including developers, project managers, and administrators
    • This includes bug tracking software, which can link to the bug in the repository
    • Project management tools can also link to the repository (Jira)
    • Build machines can also link to the repository (Jenkins). It can monitor for changes, compile the code, run metrics and automated tests, and package
  • Git
    • Open source distributed version control system (VCS)
    • Can be used for any text documents as well

Client / Server vs Distributed Version Control

  • Client / Server
    • Examples include Subversion and TFS
    • Repository is housed on a central server
      • Clients connect to this central server to get files and commit
    • Conflicts must be resolved before commits are allowed
  • Distributed
    • Clients clone an entire repository to thier workstation, including all files, history, etc
      • Git stores the repository in a hidden folder named .git
      • A link is maintained to the server
    • In a peer-to-peer system, all repositories are copies of each other (although there is one version that is designated as central shared)
    • Advantages
      • You can keep working with version control (making commits, branches, fetching history, etc) offline
    • Downsides
      • Cannot lock files (this isn't necessarily a downside)
      • Great for text files, but not good for binary files (lots of version control systems have problems with these)
      • Not user friendly - hard to learn, but easy to use once you understand it
