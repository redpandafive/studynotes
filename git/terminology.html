Terminology

  • Repository
    • Also called Repo for short
    • This is the virtual store for your project
    • Contains snapshots of your code
    • Local repository
      • The repository located on your workstation
    • Remote repository
      • A repository that sits on a server that you push too
      • Can be a service, like GitHub
  • Commit
    • Creates a commit object in the repository object database
    • Moves the head pointer to point to this commit
    • HEAD remains pointed at this head pointer
  • Snapshot
    • Capture of the source code project at a moment in time
    • Must use the add command, then commit
  • Index
  • Branch
    • Lightweight, movable pointer at a specific point in time
    • Master branch is the default one created
    • Changes are made to a specific branch
  • Merge
    • Can be difficult since two branches may change the same line of code in different ways = conflict
  • Conflict
    • Occurs when two branches are trying to change the same line of code in different ways and you are attempting to merge them
  • Integration Phase
    • Combining code from multiple developers into a single code base
    • Without version control, this is very difficult since all differences / compatibility issues must be resolved
    • Should do as frequently as possible since conflicts are more likely to be smaller and easier to fix
  • Porcelain commands
    • These are commands that are most commonly used
  • Plumbing commands
    • Low level commands
    • Typically are not intended for regular use from the CLI
    • Meant to be used in scripts or tools
