Storage Accounts

  • Services that comprise Azure storage are managed through a common Azure resource called a storage account
    • Represents a shared pool of storage that can contain blob containers, file shared, tables, etc
  • Contains all of your Azure Storage data objects
  • This is an Azure Resource Manager resource
  • Provides a unique namespace for Azure Storage data
  • Naming restrictions
    • Be between 3 and 24 numbers and/or lowercase letters
    • Must be unique within Azure

Associated Resources

Storage Accounts

  • Gives a unique namespace in Azure
    • Each object you store has an address that includes your unique account name
    • Account name + Blob Storage endpoint forms the base address for the objects
    • Example: for storage account myaccount, http://myaccount.blob.core.windows.net

Container in a Storage Account

  • See Containers

Blobs in the Container

  • Naming rules
    • Contain any combination of characters
    • Max 1,024 characters
    • Must be case sensitive
    • Must escape reserved URL characters
    • Avoid ending with . or /
  • Example URI is simlar to https://myaccount.blob.core.windows.net/mycontainer/myblob

Types

  • There are four Types
    • Standard general purpose v2 - standard account type that is recommended for most scenarios
    • Premium block blobs
    • Premium file shares
    • Premium page blobs
  • You cannot change to a different type after it is
  • See https://learn.microsoft.com/en-us/azure/storage/common/storage-account-overview?toc=%2Fazure%2Fstorage%2Fblobs%2Ftoc.json

Storage Account Endpoint

  • Storage account provides a unique namespace in Azure for data
    • Combination of Storage Account name and service endpoint forms the endpoint for your storage account
  • Have two endpoint types available
    • Standard = 250 storage accounts per region with standard endpoints in a subscription (recommended)
    • Azure DNS zone = up to 5000 storage accounst per region (in preview)

Standard Endpoints

  • Format for the standard endpoints for each Azure Storage services
    • Blob Storage = https://myaccount.blob.core.windows.net
    • Static website (Blob Storage) = https://myaccount.web.core.windows.net
    • Data Lake Storage Gen2 = https://myaccount.dfs.core.windows.net
    • Azure Files = https://myaccount.file.core.windows.net
    • Queue Storage = https://myaccount.queue.core.windows.net
    • Table Storage = https://myaccount.table.core.windows.net

Creating

  • Since it is a Azure Resource Manager resource, it must belong to an Azure resource group
  • You can create a new resource gruop or use an existing one when creating a storage account
  • Steps
    • Select Storage Accounts from left panel
    • Click Create
    • See specifics for options here: https://learn.microsoft.com/en-us/azure/storage/common/storage-account-create?toc=%2Fazure%2Fstorage%2Fblobs%2Ftoc.json&tabs=azure-portal
