Containers

Container in a Storage Account

  • Organizes a set of blobs like a filesystem
  • Account can have unlimited containers
  • Containers can contain unlimited blobs
  • Container names must be a valid DNS name since it part of the unique URI to address the container
    • Between 3 and 63 characters long
    • Must start with letter or number
    • Can onl contain lowercase letters, numbers, and dash character (no consecutive dashes)

Creating

  • Choose Storage Accounts from left hand panel and choose the storage account
  • Select Containers from the Data Storage section
  • Click + Container and provide a name
  • Set Public Access Level to recommended level of private

Properties

  • Click the ... button next to the container, then select properties
  • The Edit Metadata option allows you add custom key/value pairs
    • Useful for organization if you hae a large number of objects
    • These do not impact how container behaves

Container Access

  • See https://learn.microsoft.com/en-us/azure/storage/blobs/blob-containers-portal

Leases

  • Use to establish or manage a lock for delete operations
  • See https://learn.microsoft.com/en-us/azure/storage/blobs/blob-containers-portal

Deleting

  • Deleting a container will delete all blobs in it as well
  • Recommended to enable soft delete so containers can be restored if needed
