Useful Terminal Commands

  • cat
    • Reads one or more files and prints their contents to standard output
    • If multiple files are listed, they are read and output in the order they appearin the command line arguments
    • cat xxxx yyyy will read and print the output for xxxx and then yyyy
  • echo
    • Prints test back to the console
    • Can redirect the output to a file instead: echo "Some Test" >> newfile.txt
  • pwd
    • Print working directory
    • See which folder you currently are in
  • cd ~
    • Go to the home directory
  • clear
    • Clears the output from the CLI
  • touch
    • Create an empty file
  • echo "SomeTestOutput" >> filename.txt
    • Prints the output "SomeTestOutput"
    • Output is directed to a filename called filename.txt
  • ls
    • Lists the content of the current directory
    • Use ls -a to see all files, including hidden ones

Team Coding Tips

  • Feature flags
    • These are simple conditional statements that test a boolean flag
      • When true, the new feature code is executed
      • When false, the old feature code is executed
    • These give you a centralized way to turn on/off new features in case of an error when deployed
      • Makes it easy to have buildable code on each commit, particuarly if the feature is still a work in progress
      • If left in for the initial release, can just flip the flag to false to rollback a feature off the new feature instead of recompiling and reloading
    if(flag) { // run new code here } else { // run existing code here }"
  • Commented Out Code
    • This can be confusing - why was it commented out?
    • This costs time for everyone who reads it trying to figure out why it is there
    • Git will remember this code block if it was committed, so don't let it pollute the code base
