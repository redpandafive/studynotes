Grid Layout

Basics

  • Offers a grid based layout system with rows and columns without resorting to floats, positioning, and tables
  • float, clear, vertical-align has no effect on grid items

Basic Example

  • The following creates a 3 x 3 grid
  • The number of parameters in the grid-template-columns rule will determine the number of columns in grid
    • Child nodes are automatically assigned to cell in the order they are written
      • 1 is assigned to column 1, row 1
      • 2 is assigned to column 2, row 1
      • 3 is assigned to column 3, row 1
      • 4 is assigned to column 1, row 2
      • ...
    • Setting all columns to auto evenly distributes the column width
  • The direct children of the grid-container element will be the cells
  • If the child elements did not contain content and no height/width specifications, the grid would be empty
    • Adding width and/or height to the child elements would show the grid
.grid-container {
display: grid;
grid-template-columns: auto auto auto;
gap: 10px;
}
.grid-item {
height: 50px;
width: 50px;
background-color: green;
}

Terminology

  • Container
    • The element that establishes the grid and wraps the grid items
  • Grid Item
    • Child elements inside a grid container (first generation children)
  • Grid Lines
    • Horizontal and vertical lines that divide a grid
  • Grid Tracks
    • Any grid column or row
    • What is between grid lines
  • Grid Cell
    • A single cell
    • Where column and row intersects
  • Grid Area
    • Any rectuangular area of one or more cells bound by 4 grid lines
  • Gap
    • The spacing between grid cells
    • Grid cells do not have margins, use gap instead

Measurement Units and Options

  • Fr Unit
    • Flexible length value
    • Represents a fraction of any space left over in a grid container
    • Calculated based on available space left over after any other layout factors that take priority
    • Useful on grid-template-rows and grid-template-columns
  • auto

    Repeat Function

    • Provides a compact way to write a recurring pattern
    • Can be used in the CSS grid properties grid-template-columns and grid-template-rows
    • Takes two arguments
      • Repeat count
        • The number of times that the track list should be repeated
        • Can be an integer value of 1 or more
        • Can also be auto-fill or auto-fit = repeats the set of tracks as many times as is needed to fill the grid container
      • Tracks
        • Specifies the set of tracks that will be repeated
        • This is one or more value, where each value represents the size of the track
    • repeat(3, auto)
      • This will repeat the second parameter three times

    Grid Container Level Rules

    Initializing the Grid

    • display: grid
      • Turns a parent element into a grid
      • All direct children automatically become grid items
      • Generates a block level grid
    • display: inline-grid
      • Turns a parent element into a grid
      • All direct children automatically become grid items
      • Generates an inline level grid

    Default Dimensions of Cells

    • grid-auto-rows: 100px
      • Sets the default height of the child cells
    • grid-auto-columns: 100px
      • Sets the default width of the child cells

    Row Definitions

    • grid-template-rows: 100px 100px 100px
      • Specifies the track height and number of row grid tracks
      • Specifies the grid should have three rows, each 100px wide
    • These accept
      • Measurement values
      • none keyword (default)
      • auto keyword
      • fit-content() function
      • minmax(min, max) function
      • maxcontent keyword
      • mincontent keyword
    • Examples

      Column Definitions

      • grid-template-columns: 100px 100px 100px
        • Specifies the size and number of column grid tracks
        • Specifies the grid should have three columns, each 100px wide
      • Use to define column count size
      • Use the repeat(n, p) to repeat measurement p for n times
      • These accept
        • Measurement values
        • none keyword (default)
        • auto keyword
        • fit-content() function
        • minmax(min, max) function
        • maxcontent keyword
        • mincontent keyword
      • grid-template-rows: 100px 100px 100px
        • Specifies the size and number of row grid tracks
        • Specifies the grid should have three rows, each 100px wide
      • grid-template-columns: 100px 100px 100px
        • Specifies the size and number of column grid tracks
        • Specifies the grid should have three columns, each 100px wide
      • Examples
        • grid-template-columns: auto auto auto
          • Specifies the grid should have three columns, each the same width and filling 1/3 of available space
        • grid-template-columns: repeat(3, auto)
          • Alternate syntax to grid-template-columns: auto auto auto

      Gap Rules

      • grid-column-gap: 5px
        • Sets the size of the gap between an element's columns
        • Above adds a 5px gap between columns
      • grid-row-gap: 5px
        • Sets the size of the gap between an element's rows
        • Above adds a 5px gap between rows
      • gap
        • Shorthand for the row-gap and column-gap properties
        • Set each independent of each other with gap: 50px 100px;
        • Set both to the same value with gap: 50px;

      Cell Level Rules

      Column Placement Rules

      • These allow you to specify the number of columns a cell should span
        • Parameters are NOT zero based
      • grid-column-start
        • Specifies which column and item in the grid should start
        • Parameter is an integer representing the column to start
        • grid-column-start: 2 starts at the second column
        • Can alternately take a span parameter (see example)
      • grid-column-end
        • Specifies at which column the item in the grid should end
        • Parameter is an integer representing the column to end before
        • grid-column-start: 3 means it should end just before the 3rd column

      Row Placement Rules

      • These allow you to specify the number of rows a cell should span
        • Parameters are NOT zero based
      • grid-row-start
        • Specifies a grid item's start position within the grid row
        • Parameter is an integer representing the column to start
        • grid-column-start: 2 starts at the second column
        • Can alternately take a span parameter (see example)
      • grid-row-end
        • Specifies which row and item in the grid should end
        • Parameter is an integer representing the row to end before
        • grid-row-start: 3 means it should end just before the 3rd row
      Examples
      Example 1: Sets an element to span two rows
      .my-target {
      grid-row-start: 1;
      grid-row-end: 3;
      }
      Example 2: Sets an element to span two rows using span syntax
      .my-target {
      grid-row-start: span 2;
      }

      Grid Areas

      • Alternative syntax to setting up a grid as columns and rows

      Setting up Areas

      Start by assigning area names to elements using grid-area CSS rule
      .item1 {
      grid-area: header;
      }
      .item2 {
      grid-area: menu;
      }
      .item3 {
      grid-area: main;
      }
      .item4 {
      grid-area: right;
      }
      .item5 {
      grid-area: footer;
      }
      Use the grid rule on the grid container to describe how the areas should be distributed
      • Takes a parameter of n string values, where n is the nubmer of rows in the grid layout you are creating
      • Each string will contain the name of the areas seperated by spaces
      • Each area name is a column
      .grid-container {
      display: grid;
      grid:
      'header header header header header header'
      'menu main main main right right'
      'menu footer footer footer footer footer';
      }
      Header
      Menu
      Main
      Right
      Footer
      • Examples
        • grid-template-areas: "Header Header Header" "Content Content Sidebar";
          • Creates a 3 column by 2 row grid
          • First row spans 3 columns and is the element tagged with the header class
          • Second row has content spanning 2 columns, then sidebar spanning 1 column

      Grid Justify and Align Rules

      • By default, items will start at the top left of the container
      • Properties that are applied to the entire grid container level
        • justify-items
          • Aligns grid items along the row axis
          • Possible values
            • start
            • end - grid item is positioned to the top right of its containing column
            • center - grid item is positioned to the top center of its containg column
            • stretch
        • align-items
          • Aligns grid items along the column axis
          • Possible values
            • center - aligns grid items vertically to the middle of the columns
        • justify-content
          • When the grid is smaller than the space for the grid container, this is used to justify the grid along the row axis
            • start
            • end - entire grid is aligned horizaonally to the far edge of the container element
            • center
            • stretch
            • space-around
            • space-between
            • space-evenly - grid is spaced evently horizaontally within the container element
        • align-content
          • When the grid is smaller than the space for the grid container, this aligns grid content along the column axis
            • end - bottom border of the grid aligns on the bottom border of the container
            • space-evenly - rows are spaced evenly in the container
          • Used to align the grid content along the column axis
      • The following are used for items within the grid cells themselves (applied to grid item, not grid)
        • justify-self
          • Used to align a grid item along the row axis
        • align-self
          • Used to align a grid item along the column axis
