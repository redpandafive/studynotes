Pseudo Classes

  • A keyword added to a selector that specifies a special state of the selected element(s
  • Syntax consists of a color followed by a pseudo-class name
    • The element it is attached to is defined as an anchor element
  • This extends styling to an element not only in relation to the content of the document tree, but also:
    • External factors like the history of the navigator (:visited)
    • Status of its content (:checked)
    • Position of the mouse (:hover)
  • https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-classes

Functional Pseudo Classes

  • :has()
    • Matches an element if any of the relative selectors that are passed as an argument match at least one element when anchored against this element
  • :is()
    • Takes a selector list as its argument
      • This is a forgiving selector list, so any selector that cannot be parsed is ignored (instead of invalidating the entire list)
    • Selects any element that can be selected by one of the selectors in that list
    • Takes the specificity of the most specific selector in its arguments
    • T
  • :not()
    • Matches elements that do not match a list of selectors
  • :where()
    • Takes a selector list as its argument
      • This is a forgiving selector list, so any selector that cannot be parsed is ignored (instead of invalidating the entire list)
    • Selects any element that can be selected by one of the selectors in that list
    • Differs from :is() in that it alwsys has a 0 specificity

User Action Pseudo Clases

  • Require user interaction to apply
    • :focus
      • Matches when an element has focus
    • :hover
      • Matches when the mouse is over the anchor item
