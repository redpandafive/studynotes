Flexbox

Vs Grid

  • There is a lot of disagreement over when to use Flexbox vs when to use Grid
  • One mode of thinking is use
    • Flexbox for smaller stuff like aligning text to an icon
    • Grid for larger stuff like page layouts
  • However, this is not always the case. Sometimes you can accomplish the same goal with both, but the syntax may be shorter in one
    • You may limit your possibilities if you restrict yourself to this thought process
  • One dimension (Flexbox) vs Two dimension (Grid)
    • Flexbox will layout elements in one direction (either vertical or horizontal)
      • Allows a row based OR column based layout
    • Grid works along both horizontal and vertical axis
      • Allows precise placement into cells defined by rows and columns
  • Another way to think about which is the better choice is:
    • Should content shape the layout? Use Flexbox
      • Layout will adjust based on the content you are working with
      • Sometimes its hard to predice behavior at certain viewports, so use Grid if you need precision
    • Should layout shape content? Use Grid
      • For instance, the grid cells will not adapt to accomodate longer content
      • Grid focuses on precise content placement, so you if accuracy is important, use it
  • You do not have to always choose one or the other - they can be combined
