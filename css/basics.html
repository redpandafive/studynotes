Basics

Rules

  • box-shadow
    • Adds shadow effects around an elements frame
    • Described by
      • X and Y offsets relative to the element
      • Blur
      • Spread radius
      • Color
      • https://developer.mozilla.org/en-US/docs/Web/CSS/box-shadow
  • display
    • Sets whether an element is treated as a block or inline element and the layout used for its children
      • Specificaly, it sets an element's inner and outer display types
    • CSS Level 3 supports setting inner and outer display types seperately, but it is not well supported right now
      • I excluded all notes related to this: see https://developer.mozilla.org/en-US/docs/Web/CSS/display
    • Outer display type
      • Sets an element's participation in flow layout
      • Possible valueChanges
        • block
          • The element generates a block element box
          • Generates line breaks both before and after the element when in the normal flow
        • inline
          • The element generates one or more inline element boxes that do not generate line breaks before or after themselves
          • In normal flow, the next element will be on the same line if there is space
    • Inner display type
      • Sets the layout of children
      • Possible values
        • table - behaves like a table element
        • flex - behaves like a block element and uses flexbox model
        • grid - behaves like a block element and uses grid model
    • Box display type
      • Possible values
        • contents
          • Element doesnt produce a specific box by themselves
          • Element is replaced by their pseudo-box
        • none
          • Turns off display of element and all its children
          • Element has not effect on layout (unlike visibility)
  • height
    • Note, this rule cannot be set on an element with display: inline
  • list-style-type
    • Specifies the marker of a list item element
    • https://developer.mozilla.org/en-US/docs/Web/CSS/list-style-type
  • transition
  • position
  • visibility
    • Shows and hides the element without changing the layout of a document
    • Use display instad to remove the element and remove it from the document layout
    • Possible values
      • visible
      • hidden
      • collapse - ???
    • https://developer.mozilla.org/en-US/docs/Web/CSS/visibility
    • >
  • width
    • Note, this rule cannot be set on an element with display: inline
  • white-space
    • Sets how white space inside an element is handled
    • Specifies if and how white space is collapsed and whether lines may wrap at soft wrap opportunities
    • https://developer.mozilla.org/en-US/docs/Web/CSS/white-space

Pseudo Selectors

  • :empty
    • Select an element that does not contain any children, including:
      • Element nodes
      • Text
      • Whitespace
    • This does not consider comments to be content
  • :Host
    • Useful in Angular to reference a component template element from within itself
