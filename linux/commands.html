Commands

Commands

alias

  • Creates an alias of a command
  • Allows you to create shortcuts for longer commands
  • alias .3="cd ../.." = you can now use .3 to execute cd ../../..
  • To make the aliases consistent across reboots of your computer, you must add them to your .bashrc or .bash_aliases file.

cat

  • Reads one or more files and prints their contents to standard output
  • If multiple files are listed, they are read and output in the order they appearing the command line arguments
  • cat xxxx yyyy will read and print the output for xxxx and then yyyy

cd

  • Change the current working directory
  • You can supply absolute or relative filepaths
  • cd .. = move to the parent directory of the current working directory
  • cd ../.. = move to the grandparent directory of the current working directory
    • You can add on as many additional ../ as you work to move up higher
  • cd ~ = go to the home directory from anywhere in the filesystem
  • cd - = changes the current working directory back to the one you just came from
    • This allows you to go back and forth between two directories
  • cd dev/mobile/android = change the current working directory to the specified path starting with dev as a subfolder

clear

  • Clears the output from the CLI

echo

  • Prints test back to the console
  • Can redirect the output to a file instead: echo "Some Test" >> newfile.txt

ls

  • Lists the contents of a directory
  • ls = Lists the content of the current working directory
  • ls Help gc_help = Lists the contents of the Help directory first, then the contents of the gc_help directory
  • Use -l for a long listing
    • ls -1 = Lists the content of the current directory with all output in a single column
    • Breakdown of output
      • Line 1 = total size of all files in the listing
      • Line 2 = = each file or directory is on a line by itself
      • For each line
        • 1st character is the file type
          • - = regular file
          • b = block special file
          • c = character special file
          • d = directory
          • l = symbolic link
          • n = network file
          • p = named pipe
          • s = socket
        • Next 9 characters are 3 groups of permissions
          • Permissions for file owner (1-3)
          • Permissions for group (4-6)
          • Permissions for others (7-9)
          • Within each group, there will be three columns
            • 1st column = Read permission column
              • Will contain r or -
            • 2nd column = Write permission column
              • Will contain w or -
            • 3rd column = Execute permission column
              • Will contain x or -
        • Number of hard links to the file or directory
          • Files are usually 1, unless other hard links are created
          • Directories usually have at least 2, one is the link itself and the other is its entry in the parent directory
        • Name of the owner
        • Group
        • File size
        • Date of last modification
        • Filename
  • ls -a = use to see all files, including hidden ones
  • ls -A = almost all option, displays the same as all output files, but with the "." and ".." entries
  • ls -d = look at a directory
  • ls -h = human readable format that gives file sizes in most appropriate units
  • ls -l = long listed
  • ls -hl
  • Pattern matching can be done to selectively lists a set of files
    • ? represents any single character
    • * represents any string of characters
    • ls *.c = find all files with .c extensions
    • ls ip_* = list any files or directories that have names starting with "ip_"
  • Use the hide option to omit certain files form a listing
    • ls --hide=*.bak = removes all files with a .bak extension from the listing

pwd

  • Print working directory
  • See which folder you currently are in

tree

  • Print the directory tree from inside the current working directory
  • This will display all descendant file and folders

touch

  • Use to create an empty file

Other

Absolute vs Relative Filepaths

  • Absolute paths are:
    • Prefixed with a / character
    • Start at the root
    • For example, ls -d /dev will start at the root directory and look for a "dev" there
  • Relative paths are
    • The current working directory is the "root" or base directory for relative paths
    • For example, ls -d dev will start at the current working directory and look for a "dev" there

Output Redirect

  • echo "SomeTestOutput" >> filename.txt
    • Prints the output "SomeTestOutput"
    • Output is directed to a filename called filename.txt
