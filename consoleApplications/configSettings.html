Configuration Settings

  • Allow changing the behavior of the application without changing and recompiling code

.NET

  • Done with an app.config file
  • Drawbacks
    • Loosely typed - values must be parsed, usually by a TryParse function
    • Since all configurations are stored in a single file, this can lead to bloat
    • No configuration hierarchy that allows for overrides

.NET Core

  • Settings are defined in a appsettings.json file
  • Requires the following extension packages, which you can install via NuGet
    • Microsoft.Extensions.Configuration
    • Microsoft.Extensions.Configuration.FileExtensions
    • Microsoft.Extensions.Configuration.Json
    • Microsoft.Extensions.Configuration.Binder

Appsettings.json Example

Define an appsetting.json file with desired settings
{
"character": {
"firstName": "Harvey",
"lastName": "Birdman"
}
}
Define a class to represent the configuration values in code
public class Person
{
public string FirstName { get; set; }
public string LastName { get; set; }
}
var builder = new ConfigurationBuilder()
.SetBasePath(Directory.GetCurrentDirectory())
.AddJsonFile("appsettings.json");

var config = builder.Build();
var author = config.GetSection("character").Get()
