Disable Autocomplete on a Form Control

  • Browsers remember information that the user smites through input fields on websites
    • This allows autocompletion (suggest possible completetions) or autofill (pre-populate fields on load)
  • This attribute allows the author to indicate the field will not be useful in the future (such as a one time pin) or contains sensitve information
  • This has two effects
    • Tells the browser not to save data inputted for later use on similar forms
      • However, browser standards do not require this behavior
    • Stops the browser from caching form data in the session history
  • Modern browsers do not support autocomplete="off" for login fields
    • This is done to allow account managers to store user credentials
  • For something like a user management page where you aer specifying a password, you can use autocomplete="new-password" to tell the browser not to autofill the password value
To disable at the control level
<input type="text" id="cc" name="cc" autocomplete="off" />
To disable at the form level
<form method="post" action="/form" autocomplete="off">
