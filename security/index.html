Spammer Traps ("honey pots")

  • One technique to deter spambots without resorting to captchas or other hoops
  • Adds an invisible control to forms
    • Bots will parse the HTML and see the control and automatically fill in some value
    • Users cannot see it, so they will not submit a value in this form
    • On sever side, check the value of this control. Reject the request if there is a value there since you can assume it is a spambot
      • Log it in case span does get through so you can see if additional modifications are needed
  • Advanced tips
    • Use a realisting sounding name on the control that you do not use, like phone or website
    • Use JS to hide the control since CSS can be parsed by bots
    • Use a form timeout to prevent instant posing
      • For instance, initially disable the submit button, but enable it after 5 seconds
    • Disable autocomplete on the control
