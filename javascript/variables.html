Variables and Constants

  • Variables and constants must be declared before they can be used
  • Type is decided when you assign a value to a variable
    • You can switch the type of a variable by assigning a different value type, but this is generally discouraged

Pre ES6

  • Only way is to use the var keyword
    • <

ES6 and Later

  • Uses the let and const keywords
  • See block section of Scope
    • Only defined within the block of code in which they appear
  • Can declare multiple variables in one let statement: let i, sum;
  • Good practice to set an initial value on the same line
    • Variable will contain undefined if not
    • Constants must be initialized when you declare it
  • One coding convention capitalizes all letters in a constant identifier

Scope

  • Global scope
    • Declarations made at the top level outside of any code blocks
    • In Node and client side JS modules, the scope is the file it is defined in
    • In traditional client side JS, the scope it the HTML document in which it is defined
      • A global declaration declared in one script, it is defined in all of the script elements in the document
  • Block scope
    • Only defined within the block of code in which they appear
    • Cannot delcare 2 variable or constants with the same name in the same block
      • You can do this if one is in a nested scope, but this is generally discouraged
  • Function scope
    • Variables are scoped to the containing function
      • Nesting within the function does not make a differece
      • Global variables are declared as properties on the Global object
      • It is legal to declare the same variable multiple times
