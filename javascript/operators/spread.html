Spread Syntax

  • Not a true operator in the sense that it cannot be evaluated to produce a value.
  • Can destructure an array and expands it into individual elements
    • Unpacks (or spreads out) the elements of an array in a context where individual values are expected
    const originalArray = [1, 2, 3, 4, 5]; const newArray = [...originalArray.slice(0,3), 6, ...originalArray.slice(3)]] console.log(newArray); // outputs [1, 2, 3, 6, 4, 5]
  • Can be used with an object to create a new object with a subset of properties from another object
    • const originalObject = {a: 1, b: 2, c: 3}; const newObject = {...originalObject, b: 4}; console.log(newObject); // outputs {a: 1, b: 4, c: 3}
  • Other applications
    • Make shallow copies of JS objects
    • Append one array to another
    • Create a copy of an array
    • Separate arguments in a function invocation
      • Rest Parameters have a similar syntax, but can be used in a function definition to gather multiple function arguments into an array
      • This is the opposite effect to the spread operator
  • Be careful when using this heavily since it is a substantial amount of work to the JavaScript interpreter
  • Can be used on any iterable object
  • New syntax added in ES6

Examples

To spread elements of an array to function arguments
function multiply(number1, number2, number3) {
console.log(number1 * number2 * number3);
}
let numbers = [1,2,3];
multiply(...numbers);
To copy an array
let array2 = [...array1];
To append an array to another without modifying the original arrays
let desserts = ['cake', 'cookie', 'donut'];
let desserts1 = ['icecream', 'flan', 'frozen yoghurt', ...desserts];
