Operators

  • Operators combine subexpressions into new expressions
  • Expressions are evaluated left to right (this is not the same thing as operator precedence)
  • In w = x + y * z
    • Subexpression w is evaluated first
    • Subexpressions x, y, and z are evaluated
    • Values of y and z are multiplied
    • Above is added to value of x
    • Above is assigned to the variable or property specified by expression w

Operator Categories

  • Unary
    • Converts a single expression into a single, more complex expression
    • Negation operator (-) and delete operator are examples
  • Binary
    • Combine two expressions into a single, more complex epxression
    • Includes most operators
    • Ternary
    • Combines three expressions into a single expression
    • Conditional operator (?:) is teh only one

Operator Associativity

  • The order that operators of the same precedence are performed
  • Left-to-right associativity
    • Operations are performed from left to right
    • w = x - y - z; is equivalent to w = ((x - y) - z);
  • Right-to-left associativity
    • Operations are performed from right to left
    • w = x = y = z; is equivalent to w = (x = (y = z));
    • q = a?b:c?d:e?f:g; is equivalent to q = a?b:(c?d:(e?f:g));

Operand Types and Operators

  • Some operators require their operands to be a specific type
  • Some can work with any type
    • Every JS value is either truthy or falsy, so operators that expect boolean operands will work with an operand of any type
  • Operators usually convert the type of their operands as needed
    • "3" * "5" results in 15. The multiplication operator converts both strings to numbers automatically.
  • Some operators behave differently, depending on the type of operands used
    • The + operator will add numeric operands togehter, but concatenate string operands

Operator Side Effects

  • Some expressions have side effects, so their evaluation may affect the result of future evaluations
  • Assignment operators are good examples of these

Arithmatic Operators

  • **, *, /, %, +, -
  • ** is a new operator called the exponentiation operator
  • All numbers are floating point, so all division operations have floating point results

Unary Operators

  • + as a unary operator converts its operand to a number and performs no action on a number
  • -, ++, --

Testing for Equality

  • Equality operator (==) applies type conversions, which can lead to undesired results
    • "0" == false => true since both are converted to zero
  • Strict equality operator (=== and !==) does not consider operands to be equal if they are not of the same type

in

  • Left side should be a string (or something that can be converted to one)
  • Right side should be an object
  • Returns true is the left side value is the name of a property of the right side object
// An array with elements (indices) 0, 1, and 2
let data = [7,8,9];
// => true: array has an element "0"
"0" in data
// => true: numbers are converted to strings
1 in data
// => false: no element 3
3 in data

instanceof

  • Left side should be an object
  • Right side should be a class of objects (the constructor function that creates the class)
  • Returns true is the left side value has the right hand function is in its prototype chain
  • See Classes for more details
