Arrays

  • Specialized form of a JS object
    • Array indexes are really little more than property names that happen to be integers
    • However, they are optimized so access to indexes is generally significantly faster than access to regular object properties
    • They have the following features other objects do not have
      • Length property is automatically updated as elements change
      • Setting the length property to a smaller value truncates the array
      • Inherits useful methods from Array.prototype
      • Array.isArray() returns true for it
  • For associative arrays, see Maps
  • Represent an ordered collection of values
    • Values in the array are called elements
    • Each element has a numeric position in the array, known as its index
    • Index numbers are 32 bit and can range from 0 to 4,294,967,294
  • Untyped
    • Can contain any type, including a objects or other arrays
    • Can be mixture of types
    • ES6 adds new array classes that can be typed
  • Dynamic
    • Grow and shrink as needed automatically
    • No need to declare a fixed size or reallocate it if more elements are needed
  • Can be sparse
    • Elements need not be contiguous, so you can have gaps
    • In a sparse array, the value of the length property will be greater than the number of elements
    • See Filter() for an example of removing gaps from a sparse array
