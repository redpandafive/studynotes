Creating Objects

Object Initializers / Object Literals

  • Expressions whose value is a newly created object
    • Are not primary expressions
  • Also known as object literals
  • Composed of
    • Property name = a JavaScript identifier or string literal
      • Must be quoted if using a space or hyphen in the name
    • Property value = any JavaScript expression
      • The value of the expression is the value of the property
An object with 2 properties
et p = { x: 2.3, y: -1.2 };
ES6 allows nested object literals
let rectangle = {
upperLeft: { x: 2, y: 2 },
lowerRight: { x: 4, y: 5 }
};

Object Creation Expressions

  • Creates a new object
  • Invokes a constructor function to initialize the properties of that object
  • If no arguments are passed, no parentheses are needed
    • The entire result will be undefined
    • Will not throw a TypeError
new Object();
new Object;
new Point(2,3);

Object.create()

  • Creates a new object using its first argument as a prototype of that object
    // o1 inherits properties x and y.
    let o1 = Object.create({x: 1, y: 2});
    // results in 3
    o1.x + o1.y
  • Passing null will create a new object that has no prototype and no inherited properties, not even basic things like toString
    let o1 = Object.create(null);
  • Instead, pass Object.prototype to get an object like {} or new Object()
    let o1 = Object.create(Object.prototype);
  • Useful when you want to pass an object, but want to prevent modification of it
    • The created object will inherit all properties of the original
    • Changes to the new object will not alter the original object
