Freezing Objects and Arrays

  • While const prevents you from adding or removing properties from an object, it does not prevent you from modifying the value of properties
  • The freeze method is a static method on the Object class that returns an object that cannot be modified, including the values of its Properties
    • Freezing an object is the highest integrity level that JavaScript provides.
  • If you attempt to modify a property in:
    • Non-strict mode - the property value will not change, but not warning or error will be displayed
    • Strict mode - an error will be thrown
  • Prevents extensions and makes existing properties non-writable and non-configurable
    • New properties cannot be added
    • Existing properties cannot be removed
    • Their enumerability, configurability, writability, or value cannot be changed
    • The object's prototype cannot be re-assigned
  • Calling freeze() on it again returns the same object
  • This is a must have if you are doing functional programming with JS to make your objects immutable

Nested Objects

  • Calling freeze on an object will not freeze any nested objects
  • Call freeze on any nested objects to get a fully immutable object

Arrays

  • Arrays can be frozen using this method as well
  • Syntax is the same as objects

Examples

const person = Object.freeze({ name: 'Bob', age: 35, city: 'New York', state: 'NY' })
Call freeze on any nested objects to make them immutable
const person = Object.freeze({
name: 'Bob',
age: 35,
city: 'New York',
state: 'NY'
})
Call freeze on any nested objects to make them immutable
const person = Object.freeze({
name: 'Bob',
age: 35,
address: Object.freeze({
city: 'New York',
state: 'NY'
})
})
Freeze arrays
const hobbies = Object.freeze(["Water Skiing", "Video Games", "Car Racing"])
