Prototypes and Prototype Inheritance

  • JS uses the term "own property" to refer to non-inherited properties
  • Almost every JS object has a second JS object associated with it known as a prototype
    • Objects inherit the properties from its prototype
  • All objects created by object literals have the same prototype object
    • Use Object.prototype to refer to this object
  • Objects created with the new keyword and a constructor invocation are different
    • The value of the prototype property of the constructor function is their prototype
    • Object created by new Object() inherits from Object.prototype (like {})
    • Object created by new Array() inherits from Date.prototype (like {})
  • Prototype chain = linked series of prototype objects
  • The prototype chain is searched for looking for a property
    • When querying for a property on an object x, it first checks "its own properties"
    • If not found there, the prototype object of x is checked for the property.
    • If not found there and there is a prototype object, check that for the property
    • This continues until the property is found or an object with a null prototype is searched
  • Inheritance occurs when querying properties, but not setting them
    • Making an assignment to a prototype property creates a property on the outermost object
    • It will never make an assignment to a prototype property.
    • This allows us to selectively override inherited properties.
    • If you add a property to an object that has the same name as a property on the prototype, it only hides the property on the prototype.
  • Inheritance is dynamic
    • Properties modified after creation are still inherited
    • Useful for adding new language features in older versions of the language
      • Must be careful with this since it can lead to future incompatibility and confusion
      • Could even add to Object.prototype, but this usually is not a good idea
