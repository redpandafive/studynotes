Declarations and Variables

  • Variables and constants must be declared before they can be used
  • The type is decided when you assign a value to a variable
    • You can switch the type of a variable by assigning a different value type, but this is generally discouraged

Identifiers

  • Identifiers must follow these rules
    • Can begin with a letter, underscore, or dollar sign
    • Subsequent characters can be letters, digits, underscores, or dollar signs
  • Reserved words cannot be used as identifiers
    • Some are only reserved in certain contexts
    • Some cannot be fully reserved to keep backwards compatibility with older programs (such as let)
    • Some reserved words are restricted since they might be used in the future, such as interface, public, and protected
  • JS does allow Unicode letters, digits, and ideographs in identifiers
    • The typical convention is to stick with ASCII characters in identifiers
    • Unicode escape sequences can be used for incompatible systems
  • There are different code conventions for identifiers
    • One coding convention capitalizes all letters in a constant identifier

Pre ES6

  • Only way is to use the var keyword

ES6 and Later

  • Use the let and const keywords
    • Replaces var, so no reason to use that keyword now
  • These follow block scoping
    • Only defined within the block of code in which they appear
  • If you attempt use a variable declared with these keywords without initializing it, you will get an error
    • Contrast this to var which will just generate an undefined value
  • Multiple variables can be declared in one let statement: let i, sum;
  • Good practice to set an initial value on the same line as the declaration
    • Otherwise, the variable will contain undefined
    • Constants must be initialized when you declare it
