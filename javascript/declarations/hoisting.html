Hoisting

  • Variables declared anywhere in the function are hoisted to the top of the function
    • Allows them to be used anywhere in the function
  • The initialization remains where you wrote it
    • If you reference the variable before the initialiaiton code, it will contain undefined
    • This will not be flagged as an error, so it is a source of a lot of bugs
  • An unusual feature of using var and function scope
