Statements

  • While expressions are evaluated to produce a value, statements are JavaScript sentences or commands that make somethign happen
  • JS programs are just a sequence of statements that are executed in a default order
    • Default is the order they are written
    • This order can be altered by a control structure
      • Conditionala
      • Loopa
      • Jumps
  • An expression with side effects (assignments and function invocations) can be a statement
    • This is called an expression statement
  • Declaration statements
    • Declare new variables and define new functions
  • Terminated with semicolons

Expression Statements

  • Simplest kind of statement that is an expression with side effects
  • Assignment Statements
    • ++ and -- operators have the side effect of changing a variable value, just as if an assignment has been performed
    • delete operator deletes an object property and is almost always used as a statement, rather than part of a larger expression
  • Function Invocations
    • Function calls are expressions, but they have side effects that affect the host environment or program state
    • A funciton call without a side effect (such as returning a result that is assigned to a variable) is pointless, so they are also considered statements

Compound Statements

  • A statement block combines multiple statements into a single compound statement
    • They are sequence of statements enclosed within curly braces
    • Act as a single single statement and can be used anywhere JS expects a single statement

Empty Statements

  • Simply a semicolon
  • The JS interpreter takes no action when this is encounterd
  • Usually a good idea to put a commnet in your code when you need to use one of these
