Jump Statements

  • Cause the JavaScript interpreter to jump to a new location in source code

Break

  • Jump to the end of a loop or switch statement
  • Use in loops to end an iteration without executing all the code
  • Can includes an optional label that allows the program to break out of a labelled statement
    • Break statements must be nested within any label it references
      outerBlock: {
      innerBlock: {
      console.log("1");
      break outerBlock; // breaks out of both innerBlock and outerBlock
      console.log(":-("); // skipped
      }
      console.log("2"); // skipped
      }

Continue

  • Skip the rest of the body of a loop and jump back to the top of a loop to begin a new iteration
  • This restarts the loop at the next iteration
  • Can includes an optional label similar to break

Return

  • Jump from a function invocation back to the code that invoked it
  • if an expression is added after the return statement, that expression is evaluated and returned to the code that invoked the function
  • If no expression is added after the return statement, undefined is returned

Throw

  • Raises an exception
  • Try/catch/finally statements can be used to handle the error
    • Try - block of code whose exceptions are to be handled
    • Catch
      • Block of code to execute when an error occurs
      • Can also include an identifier in parentheses. The exception object will be assigned here.
      • Optional
    • Finally - block of cleanup code guaranteed to be executed (optional)
  • The interpreter will jump to the nearest handler when throw raises and error
    • The interpreter will check up the call stack until it finds a catch statement
    • If none is found, the program reports an error to the user
  • Any expression can follow a throw statement
    • This can evaluate to any type
    • This could be a number indicating an error code or a string that contains a human readable message
    • Can also be an instance of the Error class or one of its subclasses
      • This includes a message (string passed to constructor function) and name (type of error) properties
