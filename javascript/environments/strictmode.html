Strict Mode

  • This is a directive introduced in ES5
  • Contains no language keywords, just a special string literal
  • Can only appear at the start of a script or function body, before any real statements have appeared
  • Code that is written in classes and modules is automatically in strict mode

Delete Operator

  • In strict mode, the delete operator will raise an error if its operand is an unqualified identifier, such as
    • variable
    • function
    • function parameter
  • An error will also raise when trying to delete a non configurable property

Eval

  • In strict mode, eval will
    • Perform a local eval with a private variable environment that is discarded when it returns
    • Can query and set local variables
    • Cannot define new variables or functions in local scope
  • Eval is treated more like a keyword
    • Cannot overwrite eval function with a new variable

Functions

  • Functions invoked as function have a this value of undefined
  • When invoked with call() or apply(), the this value is exactly the value passed as the first argument to call() or apply()
  • Cannot have two or more parameters with the same name

Properties

  • The following will trigger an error (instead of failing silently)
    • Assignments to non writable properties
    • Attempts to create new properties on non-extensible objects
  • An object literal cannot define two or more properties by the same name

Variable Declaration

  • If you attempt to use an undeclared variable, you will get a reference error when you run your code
    • Outside of strict mode, it will automatically be created as a new global variable

With

  • Forbidden in strict mode
