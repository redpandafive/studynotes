Regular Expressions

  • An object that describes a textual pattern

RegExp Objects

  • Represent a regular expression
  • Can be created with
    • RegExp constructor
    • A special literal syntax
    The following two lines are equivalent
    let pattern = /s$/;
    let pattern = new RegExp("s$");

Regular Expression Literals

  • Specified as characters within a pair of slash characters

Definition

  • Most characters (alphanumeric) describe characters to be matched literally
    • /java/ matches any string that contains the substring java
  • Some characters are not matched literally, but have special significance.
    • Example is the $ character that matches the end of the string
    • s$ matches any string that contains the letter s as its last character
  • Flag characters affect how regular expressions work
    • /s$/i matches strings that have s as the last character using a case insensitive search
  • Escape sequences can represent certain non-alphanumeric characters
    • \t = tab
    • \n = newline
    • \r = carriage return
  • Several punctuation characters have special meanings
    • These must be escaped with \ to be treated literally

Character Classes

  • Matches any one character that is contained within it
    • /[abc]/ matches any one of the letters a, b, or c
  • Negated character classes match any character that is not included in the class
    • /[^abc]/ matches any one character other than a, b, or c
  • Ranges of characters can be specified using the hyphen
    • /[a-z]/ matches any one lowercase character /[a-zA-Z0-9]/ matches any letter or digit from the Latin alphabet
  • Common shortcut character classes include
    • \s matches the space, tab, or any other Unicode whitespace character
    • \S matches any character that is not Unicode whitespace
    • \d matches any ASCII digit (same as [0-9])
    • \D matches any character other than an ASCII digit (same as [^0-9])

Repetition

  • Specify how many times an element of a regular expression may be repeated
  • Applies to the preceding single character or character class
  • Use parentheses to define a group to match on
  • Greedy vs non-greedy repetition
    • Greedy repetition

Greedy Repetition Syntax

  • Repetition syntax
    • {n,m} = match the previous item at least n times but no more than m times
    • {n,} = match the previous item n or more times
    • {n} = match the previous item exactly n times
  • Common cases have their own shorthand syntax
    • ? = matches zero or more occurrences of the previous item
      • Means the previous item is optional
      • Same as {0,1}
      • Use with caution - this is allowed to match nothing
    • + = matches one or more occurrences of the previous item
      • Equivalent to {1,}
    • * = matches zero or more occurrences of the previous item
      • Equivalent to {0,}
      • Use with caution - this is allowed to match nothing

Examples

  • /\d{2,4}/; = match between two and four digits
  • /\w{3}\d?/; = match exactly three word characters and an optional digit
  • /\s+java\s+/; = match "java" with one or more spaces before and after
