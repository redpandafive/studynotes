Control Statements

For / In Loops

  • Enumerates over all enumerable string properties of an object
    • This includes inherited enumerable properties from its prototype chain
    • Does not include symbol properties
  • To remember its usage, think "for..in keys == foreign keys == use for..in for keys"
  • You can use a for..in loop with arrays, but it is better to use for, foreach, or for of loop instead
    • These other methods avoid non index properties
  • You can use a constant declaration as long as a new value is not assigned within the loop
    for(const property in object) { console.log(property); }

For / Of Loops

In this pseudocode, variable receives a value form each iterator, iterable is an iterable object that is operated on, statement is a block statement executed for every iteration
for (variable of iterable) { statement }
  • This loop is said to iterate over the iterable, operating on each item operates one by one in sequential order
  • Loops over an iterable object, invoking the Symbol.iterator property on the iterable
    • This returns interator object
    • The loop will call the interactor's next() method to produce the values to be assigned to the variable
    • This will continue until
      • The iterator's next method returns an object containing done: true
      • A break exists the loop
      • a return statement runs
  • Returns a list of values of the object being iterated
    • Compare with for..in loops that returns a list of keys on the object being iterated 
    let list = [4, 5, 6];
                  for (let i in list) {
                    console.log(i); // "0", "1", "2",
                  }
                  for (let i of list) {
                    console.log(i); // 4, 5, 6
                  }
  • You can use a constant declaration as the variable as long as a new value is not assigned within the loop
    for(const datum of data) { console.log(property); }
