Basics

Language Characteristics

JS is
  • An interpreted programming language
  • Suitable for multiple programming styles, including object oriented and functional programming

History

  • ECMASCript is the specification name for the language
  • JavaScript is a trademark licensed from Oracle to describe the the original Netscape implementation
  • In practice, everyone just calls the language JavaScript

Lexical Characteristics

  • Syntax is loosely based on Java, but the languages are otherwise unrelated
  • Case sensitive, so identifiers must be typed with a consistent capitalization of letters
  • Largely ignores whitespace and line breaks

Host Environment

  • The core JavaScript language defines a minimal api for working with numbers, texts, arrays, etc
    • It does include any input or output functionality
    • This is the responsibility of the host environment within which JavaScript runs
  • Host environments include
    • Web browsers
      • Original and most commonly used host environment for JS
      • Allows JS code to display output to the user combined with HTML and CSS
      • Uses APIs built into the browser
    • Node
      • Host environment that give JS access to the entire OS
      • Allows functions not allowed in a browser, such as reading and writing files
      • Is used for a wide range of functions, including
        • Implementing web servers
        • Simple utility scripts to replace shell scripts
  • Garbage collection is performed automatically by the JavaScript hosting environment

Line Delimiters

  • JS uses semicolons as line delimiters
    • These are optional
    • You can usually omit the semicolon between two statements if those statements are written on separate lines
  • Generally, JS treats a line break as a semicolon if the next non-space character cannot be interpreted as a continuation of the current statement. There are three exceptions
    • A line break after certain identifiers is always interpreted as a semicolon
      • return
      • throw
      • yield
      • break
      • continue
  • There are two coding styles for adding delimiters
    • Always add a delimiter, even when it is not required
    • Only add a delimiter when it is required
  • Defensive semicolons can be added to guard against unintentional interpretation
    // defensive semicolon prevents absence of semicolon from being interpreted as a line continuation let x = 0
    ;[x,x+1,x+2].forEach(console.log)
