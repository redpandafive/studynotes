Function Parameters

  • When you invoke a function with a list of argument values, those values are assigned to the parameters declared in the function definition
    • Similar to variable assignment
  • Function definitions do not check parameters passed as arguments to a function
    • No type checking
    • Does not check the number of arguments passed

Optional Parameters

  • Allow you to pass fewer arguments than the function signature defines
  • If there are fewer arguments than the declared parameters, a default value is passed (usually undefined)
  • Optional parameters should be tested before attempting to use them
  • Always list optional parameters at the end of the parameter list
Testing with a conditional
Use a conditional
if (a === undefined) {
a = [];
}
Use the || operator as a shorthand. Returns first argument if truthy, otherwise returns the second
a = a || [];

Default Parameters

  • In ES6, you can specify a default value to use if one is not passed to the function
  • These are evaluated when the function is called, not when defined
  • Not necessary, but usually clearer to make defaults constant values
  • Can use the value of a previous parameter to define the default value of the parameters that follow it
  • Can be used with arrow functions
const rectangle = (w, h=w*2) => ({w, h});

Rest Parameters

  • Allow you to write functions that can be invoked with more arguments than parameters
  • These are called variadic functions or vararg functions
  • Precede it by three periods
  • Must be the lsat parameter in a function declaration
  • The rest parameters will be passed as an array
    • It will contain all arguments passed as non-rest parameters
    • It will never be undefined, just an empty array
    • Never useful to define a parameter default for a rest parameter
  • Don't confuse this with the spread operator, which can be used in function invocations
    • They are useful to use together, since you can pass in as many parameters are you want using the spread operator, but the function can capture any number of them in the rest parameter
  • Pre-ES 6, this was done using the Arguments object. However, you should not use this in new code and refactor it out of old code
function max(first=-Infinity, ...rest) {...

Destructuring Parameters

  • Function definitions may be easier to read if you destructure arrays or objects into more clearly named parameters in the function signature
  • Parameter names within square brackets = expect an array value to be passed for each pair of square brackets
    • Array arguments are unpacked into individually named parameters
  • Parameter names in curly brackets = expect an object
    • Simpler syntax is to match the property name to the parameter name
    • Can rename the parameter name, but the syntax is more verbose
      • Left-hand side of the colon = property name
      • Right-hand side of the colon = parameter name
  • Parameters can have defaults using the usual default syntax
Function definition that passes arrays
function vectorAdd(v1, v2) {
return [v1[0] + v2[0], v1[1] + v2[1]];
}
Function definition that passes arrays, but destructures them in the signature
function vectorAdd([x1,y1], [x2,y2]) {
return [x1 + x2, y1 + y2];
}
Function definition that passes an object and destructures its properties
function vectorMultiply({x, y}, scalar) {
return { x: x*scalar, y: y*scalar };
}
vectorMultiply({x: 1, y: 2}, 2);
}
Function definition that passes an object, destructures its properties, and remaps to different parameter names
function vectorAdd({x: x1, y: y1}, {x: x2, y: y2}) {
return { x: x1 + x2, y: y1 + y2 };
}
vectorAdd({x: 1, y: 2}, {x: 3, y: 4})
