Defining Functions

  • There are three ways to define a function
    • Function Statements
    • Arrow Functions
    • Function Definition Expressions
    • Function Constructor

Function Statements

  • Consists of
    • Function keyword
    • An identifier that names the function
      • This becomes a variable whose value is the function itself
    • A pair of parentheses around a comma separated list of zero or more identifiers
      • These are the parameter names for the function and they behave as local variables within the function body
      • A pair of curly braces with zero more JS statements inside
        • This is the body of the function
        • Executed when the function is invoked
  • Function declarations are "hoisted" - it is as if they had all been moved up to the top of whatever scope they are defined in
  • Code that invokes a function can exist in your program before any code that declares the function
  • Actually declares a variable and assigns a function object to it

Function Definition Expressions

  • Defines a function and its value is the newly defined function
  • Differ from function statements in that:
    • Appears in the context of a larger expression or statement
    • The identifier is optional
    • It is up to you to assign the newly defined function object to a constant or variable
    • They are not hoisted, so you must define them before they can be evaluated
let square = function(x) { return x * x; };

Arrow Functions

  • Shorthand syntax to define a function added in ES6
    • These are expressions, not statements
    • Does not use the function keyword
    • Does not use a function named
  • Separates the argument list from the function body using =>
  • Most commonly used when you want to pass an unnamed function as an argument to another function
  • Different from functions defined in other ways since they:
    • Do not have a prototype property, so no constructor can be used
    • Use the invocation context where they are defined instead of creating a new one
      • Useful workaround if you have one nested inside a method
      • Means you have access to this in the surrounding method
Basic syntax
const sum = (x, y) => { return x + y; };
If the body is a single return statement, you can do the following:
const sum = (x, y) => x + y;
If there is only a single parameter, you can write:
const polynomial = x => x*x + 2*x + 3;
If the body is a single return statement and returns an object literal, you must include parentheses:
const polynomial = x => x*x + 2*x + 3;
If there are not parameters, you must use an empty pair of parentheses:
const f = x => { return { value: x }; };
const g = x => ({ value: x });

Function Constructor

  • Since a function is an object, there is a constructor that can be used to create new functions
  • Syntax
    • Expects one or more string arguments
    • The last argument is a string containing the function body
      • Statements should be separated by semicolons
    • Any argument before this should be a string and represents a parameter in the function
      • If the function does not have any parameters, there would only be one string passed to the constructor (the body)
  • This creates anonymous functions
  • Allows JS functions to be dynamically created and compiled at runtime
  • Less efficient in loops or frequently called functions since they must be recompiled each time they are called
  • Does not use lexical scoping - always compiled as if they are top level functions, so variables are global
  • Will probably never need to use this
The following two are equivalent
const f = new Function("x", "y", "return x*y;");
const f = function(x, y) { return x*y; };
