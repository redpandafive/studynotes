Types

Built In Types

Primitive types
  • Include
    • string
    • number
    • boolean
    • symbol
  • While technically only objects can have methods, JS primitives behave as if they have methods
  • Immutable, including strings
  • Comparisons are done by value
Primitive types that cannot contain values
  • Immutable
  • Only values that methods cannot be invoked on
  • null
    • Indicates the absence of a value
  • undefined
    • This is the value of
      • Variables that have not been initialized
      • Return value when querying an object property or array element that does not exist
      • Returned value for functions that do not explicitly return a value
      • Value of function parameters for which no arguments are passed
  • These two can often be used interchangeably
  • Equality operator considers them equal, which strict equality operator does not
Object types
  • Mutable
  • Compared by reference (do they point to the same object)
  • Assigning an object to a variable assigns the reference
    • Must compare all properties of an object or elements in an array to see if they match
  • Includes
    • Object
    • Date
    • Array
    • String, Number, Boolean (upper case version) - these are legacy wrapper objects that do not have any good reason to use

Typeof

typeof operator to find out what data type a JavaScript variable is
  • This will return a string indicating the data type
  • const isBoolean = val => 'boolean' === typeof val;
  • Examples
    • typeof "testString" => "string"
    • typeof 9753 => "number"
    • typeof true => "boolean"
    • typeof [1,2,3,4] => "object"
    • typeof function() {} => "function"
    • typeof null => "object"
    • typeof NaN => "number"
