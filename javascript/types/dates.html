Dates

  • Dates in JS are objects
    • They have a numeric representation that gives the number of elapsed milliseconds since Jan 1, 1970
  • They inherently operate in UTC
    • This can cause problems when displaying dates or handling date inputs in the user's local time zone
    • The date object does have a toISOString() method to convert dates to a string in ISO format, but is always returns the string in UTC

Parsing Date Objects to Local Date String in ISO 8601 Format

  • The date object does have a toISOString() method to convert dates to a string in ISO format, but is always returns the string in UTC
  • One solution is to use the function below to:
    • Find the difference between the UTC time and the local time
    • Adjust the Date object to reflect the local time, based on the calculated time zone offset
    • Convert the adjusted Date object to an ISO string and extract the date portion (ignoring the time part)
    • Return string result in YYYY-MM-DD format
  • See also https://dev.to/mochafreddo/handling-local-dates-in-javascript-with-formatdatetolocaliso-5edk
formatDateToLocalISO(date: Date) {
const offset = date.getTimezoneOffset();
const localDate = new Date(date.getTime() - offset * 60 * 1000);
return localDate.toISOString().split('T')[0];
};

Parsing ISO 8601 String to Date Object

  • When passing a string of format YYYY-MM-DD to a Date object constructor, you will usually encounter an "off by one day" bug
  • Since the timezone is missing from the string, it is interpreted as UTC. However, the created Date object will be adjusted to your local timezone
    • This results in the Date object being off by one day, depending on the timezone you are in
  • One approach to fixing this is to split the date string into parts
    • Months are zero based, so you do have to subtract one from it
let b = s.split('-'); let parsedDate = new Date(+b[0], (+b[1] - 1), +b[2]);
