Map

  • Holds key-value pairs
  • Remembers the original insertion order of the keys
  • Any value can be used as either a key or value
    • This includes both objects and primitive values
    • Keys are compared by identity, so if you use an object or array as a key, it will be considered different from every other object and array
  • Keys are unique in the collection, so it may only occur once
  • Only contain entries that you put it it, nothing by default
  • It is an iterable, so you can directly iterate on it
    • Can use foreach method on the Map object
    • Can use for / of loop
  • Maps hold a strong reference to its keys
    • This prevents them from being garbage collected, even if all other references to them are gone
      • Alternative is to use a WeakMap variant (see section)
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Map

Construction

  • Can be initialized with
    • No parameters
    • An iterable object that yields two element [key, value] arrays
      • Such as an array of arrays, each with an equal number of elements
    • Another map object
let m = new Map(); // Create a new, empty map
let n = new Map([
["one", 1],
["two", 2]
]);

Iteration

  • Use a for / of loop to iterate over each element in the map
  • Iterates over key/value pairs, returning a 2 element array on each iteration ([key,value])
  • Idiomatic to use destructuring assignment to assign the key and value to separate variables on each iteration
let m = new Map([[1, "one"]]);
for(let [key, value] of m) {
key // => 1
value // => "one"
}

Properties

size
  • Returns the number of key / value pairs in the Map object

Methods

clear()
  • Removes all key-value pairs from the Map object
delete(x)
  • Removes the item indexed by key x
  • Returns true if the element was removed
  • Returns false if the element does not exist
get(x)
  • Retrieves the value identified by key x
has(x)
  • Returns true/false if the key x exists
set(x,y)
  • Sets the value of y for the key of x
  • DO NOT USE BRACKET SYNTAX
    • This will appear to work, but it is actually setting the property on the generic object
    • Other Map methods will not detect these
  • Returns the Map that is being modified, so you can chain multiple fluent calls together

WeakMap

  • A variant (not actual subclass) of the Map class
  • Keeps weak references to its key values
    • Does not prevent its key values from being garbage collected
  • Intended use is to allow you to associate values with objects without causing memory leaks
  • Differences between WeakMap and Map
    • WeakMap keys must be objects or arrays; no primitives since they are not subject to garbage collection
    • WeakMap only has get, set, has, and delete methods
    • WeakMap is not iterable
      • Does not define keys, values, forEach methods
    • WeakMap does not have a size property since the size of it can change at any time as objects are garbage collected
  • Example use would be a cache object that stores computed values for later Use
    • A normal Map would not reclaim objects over the lifecycle of the Map
    • A WeakMap can reclaim objects once their last reference has dropped off
