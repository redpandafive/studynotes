Numbers

  • Represents integers and approximates real numbers
  • Integer literals can be in one of several formats, including base 10, hexadecimal, octal, and binary
  • Floating point numbers can represented using exponential notation
    • [digits][.digits][(E|e)[(+|-)]digits]
    • 1.4738223E-32 is 1.4738223 raised to the 10 to the -32 power
  • Underscore characters can be added to numeric literals for readability let billion = 1_000_000_000;
  • Floating point numbers are approximations of an actual number
    • This can lead to unexpected precision problems
      let x = .3 - .2; // thirty cents minus 20 cents
      let y = .2 - .1; // twenty cents minus 10 cents
      x === y // => false: the two values are not the same!
      x === .1 // => false: .3-.2 is not equal to .1
      y === .1 // => true: .2-.1 is equal to .1
    • This problem shows up in any programming language that uses binary floating point numbers

Numeric Operations

  • The Math object has several functions and constants that can be useful for more complex math operations
  • ES2016 adds ** operator for exponentiation

Mathmatical Errors

JS does not raise errors in cases of overflow, underflow, or division by zero. Instead, it returns special values.
  • Infinity and -Infinity
  • Anything added, substracted, multiplied, or divided by infinity results in infinity (or -infinity)
  • Can result from
    • Overflow = result of a numeric operation is larger than the largest representable number.
    • Dividing by zero
NaN is a special not-a-number value
  • Can result from
    • Dividing zero by zero
    • Divide infinity by infinity
    • Take square root of a negative number
    • Use arithmatic operators with non numeric operantds that cannot be converted to numbers
  • Does not compare equal to any other value, including itself
    • Cannot use x === NaN to detect it
    • Use x != x or Number.isNaN(x) instead
Underflow occurs when the result of a numeric operation is closer to zero than the smallest representable number
  • JS returns 0 when this occurs
Related functions on Number
  • Number.isFinite() = returns treu if the argument is a number other than NaN, Infinity, or -Infinity
Related global functions
  • isFinite() = returns true if the argument is, or can be converted to, a finite number
  • isNaN() = returns true if the argument is a non numeric value that cannot be converted to a number

Methods

  • toExponential()
    • Convertsa number to string using exponential notation
  • toFixed()
    • Number to string conversion
    • Add a parameter to specify the number of digits after the decimal point
  • toString()
    • Defaults to base 10
