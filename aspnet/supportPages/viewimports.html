ViewImport Pages

These pages are named _ViewImports.cshtml
  • Centralizes directives that apply to Razor pages so you do not have to add them to pages individually
  • Supports the following directives
    • Tag helper management
      • @addTagHelper
        • Default file includes reference to Microsoft.AspNetCore.Mvc.TagHelpers
      • @removeTagHelper
      • @tagHelperPrefix
    • @inherits
    • @namespace
      • Specifies the namespace of the pages affected by the ViewImports file
      • Default file includes reference to ApplicationName.Pages
      • There can be only one per _ViewImports file
    • @inject
      • Supports dependency injection
    • @model
      • Use to specify the Model, but this is usually done on a page-by-page basis
    • @using
      • Makes other namespaces available without having to provide their fully qualified names when working with their types
      • Default file includes reference to the default namespace of your application
  • One is created at the root of the Pages folder by default
    • Settings applied here will affect all Razor pages in the folder hierarchy
  • You can have multiples of these in your Pages folder at different levels in the hierarchy
    • This allows you add to the top level _ViewImports file directives or override its settings
    • The @addTagHelper, @removeTagHelper, @inject and @using directives are additive
    • Other directives override each other the closer you get to the page
