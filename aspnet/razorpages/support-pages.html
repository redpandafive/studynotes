ViewImports Pages

These pages are named _ViewImports.cshtml
  • Centralizes directives that apply to Razor pages so you do not have to add them to pages individually
  • Supports the following directives
    • Tag helper management
      • @addTagHelper
        • Default file includes reference to Microsoft.AspNetCore.Mvc.TagHelpers
      • @removeTagHelper
      • @tagHelperPrefix
    • @inherits
    • @namespace
      • Specifies the namespace of the pages affected by the ViewImports file
      • Default file includes reference to ApplicationName.Pages
      • There can be only one per _ViewImports file
    • @inject
      • Supports dependency injection
    • @model
      • Use to specify the Model, but this is usually done on a page-by-page basis
    • @using
      • Makes other namespaces available without having to provide their fully qualified names when working with their types
      • Default file includes reference to the default namespace of your application
  • One is created at the root of the Pages folder by default
    • Settings applied here will affect all Razor pages in the folder hierarchy
  • You can have multiples of these in your Pages folder at different levels in the hierarchy
    • This allows you add to the top level _ViewImports file directives or override its settings
    • The @addTagHelper, @removeTagHelper, @inject and @using directives are additive
    • Other directives override each other the closer you get to the page

ViewStart Pages

These pages are named _ViewStart.cshtml
  • Code that is executed at the start of each Razor page execution
    • Common use case is to set the layout page for each Razor page
  • Affects all Razor pages in the same folder or any of its subfolders
  • These are hierarchical, so you can have them at different levels in the folder strucutre
  • Those located in subfolders will be executed after those located higher up the file system hierarchy

Partial Pages

  • Contain snippets of HTML and server side code to be included in any number of pages or layouts
  • Use to break up complex pages into smaller units
  • Automatically inherits the ViewData associated with the calling view, so it may behave differently on differnt Razor pages
  • View Components give a more powerful alternative to these in .NET 6

View Components

  • Replace ChildActions from ASP.NET MVC and give a more powerful alternative to Partial Pages
  • They do everything partials do and behave more consistently across Razor views
  • Do not inherit ViewData or model binding from the calling page
  • They depend on data passed when calling the view component
  • Drive from ViewComponent and typically have an associated Razor file for markup
