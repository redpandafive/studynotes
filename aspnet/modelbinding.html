Model Binding

  • Process where the ASP.NET pipeline takes a raw HTTP request and converts that into arguments for an action method invocation on a controller.
    • There are several ways that work without configuration
  • Model data could be sourced from several places:
    • Route values
      • Navigating to a route such as {controller}/{action}/{id} will allow binding to an id parameter
      • These are always bound
    • Query strings
      • Values from ?FirstName=Bob can be bound to a FirstName parameter
      • These are always bound
    • Body
      • Body from a post can be bound to an object
      • Always bound for complex types
    • Header values
      • Not commonly used
  • Note, Model Binders will not throw an error if an object is not bound, you will get a default object

ASP.NET 4

  • Out of the box, Model Binding could be done for different Content Types

ASP.NET Core

  • Use the [FromBody] attribute to tell ASP.NET Core to use the Content Type header of the request to decide which of the configured IInputFormatter to use for model binding
Example usage
[HttpPost]
public IActionResult Index([FromBody] Person person)
{
return Json(person);
}
