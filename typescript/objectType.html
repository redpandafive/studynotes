Object Types

  • There are three ways to create an object in TypeScript
    • Can be annonymous
      function greet(person: { name: string, age: number }) {
      return "Hello " + person.name;
      }
    • Can use an interface
      interface Person {
      name: string,
      age: number
      }
      ...
      function greet(person: Person) {
      return "Hello " + person.name;
      }
    • Can use a type aliases
      type Person = {
      name: string, age: number
      }
      ...
      function greet(person: Person) {
      return "Hello " + person.name;
      }
    • In each of the three styles written above, the parameter object must have a string property named name and a number property named age

Properties

  • Properties can be marked as readonly
    • No effect at runtime
    • Cannot be written to during type checking
    • You can still modify internal contents (like adding items to a readonly array), not it is not totally immutable
  • Optional properties can be identified by using the question mark
    • This tells the TypeScript compiler to just trust that a value will be assigned to the property as some point
    • Useful for
      • Computed properties
      • The following sets the Username property to be the primary key of the User table
      interface MyObject {
      optionalParam?: number
      }
