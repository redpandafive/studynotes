Type Aliases

  • Type aliases are a name for any type
    • Provide a way of creating new names for existing types
    • Don't define new types, but provide an alternate name for an existing type
    • Can refer to any valid TypeScript type, including primitive types
  • Created using the type keyword
  • Examples
    type MyNumber = number;
    type User = {
    id: number;
    name: string;
    email: string;
    }
    type Point = { x: number; y: number; };"

Primitive Aliases

  • Primitve types (string, number, boolean, etc.) can be aliased using type aliases for readability
    type Address = string;

Union Types

  • These allow us to describe values that can be one of several types and create unions of various primitive, literal, or complex types
  • Primitve union types can be aliases using type aliased for readability.
    type NullOrUndefined = null | undefined;
    type Transport = 'Bus' | 'Car' | 'Bike' | 'Walk';
A type alias for a union type
type ID = number | string;"

Function Types

  • Represents a function's type signature
  • Type aliases use => to specify the parameters and the return Type
    • Usually results in a shorter and easier to read format that using interfaces
      • type AddFn = (num1: number, num2:number) => number;
  • This can be done with either type aliases or interfaces

Extending

  • Type aliases can be extended, but it uses the intersection operator to do so
interface Client {
name: string;
}
Type VIPClient = Client & { benefits: string[]};
