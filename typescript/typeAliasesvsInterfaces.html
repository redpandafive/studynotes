Interfaces vs Type Aliases

  • Almost all features of an interface are available in Type, so in many cases you can choose between them freely
  • There are cases when you should use a type alias over an interface, including
    • To create a new name for a primitive type
    • To define a union type, tuple type, function type, or another more complex type
    • To overload functions
    • To use mapped types, conditional types, type guards, or other advanced type features
  • Type aliases are generally more expressive and do allow for more advnaced features

Differences

  • Interfaces can only be used with object type, but type aliases can be used to alias primitives and union types
  • Union types can be used with type aliases, but not interfaces
    • Interfaces can be used to create union types using a type alias
      interface CarBattery {
      power: number;
      }
      interface Engine {
      type: string;
      }
      type HybridCar = Engine | CarBattery;
  • Declarative merging can only be used with interfaces
  • Function types can be declared with either interfaces or type aliases, but type aliases usualy end up more readable
  • Both can be extended, but the syntax is different for each
