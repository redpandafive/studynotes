  • Limitations
    • More brittle than other testing methods
    • Slower than other type of testing
    • Adds additional dependencies to code base and knowledge required of the team
  • Best practices
    • Use Page Object Models for non-trivial tests
    • Check page is loaded before interacting
    • Use the simplest, most reliable method to locate elements in the page
    • Use configuration files to store things like servers and ports, just like any other project
    • Prefer explicit waits over Thread.Sleep
    • Tests should be independent of each other and not rely on executing them in a particular order
    • Use the test features, not necessarily to test visuals

Testing Basics

  • Author uses a naming convention for test classes as follows:
    • Class name - includes the item being tested, such as CreditCardWebAppShould
    • Method name - describes what the test will confirm, such as LoadApplicationPage
  • There are three logical phases within a Test
    • Arrange
      • Getting things setup
      • Setting up a WebDriver instance, navigate to a URL
    • Act
      • Perform actions in the browser
      • Clicking buttons, submitting forms, etc
    • Assert
      • Verify outcome of the test
      • Check HTML text, check browser URL, check browser title, etc
  • Testing types
    • Smoke test - very lightweight test to make sure there is no "smoke" coming out of the system

Ecosystem

  • Selenium IDE
    • Allows you to records automation test scripts using the browser
    • Scripts can be replayed to test websites
    • Scripts can be manually edited if needed
  • Selenium WebDriver
    • Code based
    • Usually more maintainable since you can leverage the programming language (such as factoring out common ocde into clases)
    • Testing architecture
      • Test framework, such as NUnit - use to write test code
      • Test code (e.g. C#) that you have written
        • This is independent of the language that the application is written in
      • IWebDriver API, such as WebDriver library installed from NuGet - use inside tests
      • Browser driver, such as ChromeDriver - talks to the browser and performs operations
      • Browser - uses HTTP to interact with Web Server
